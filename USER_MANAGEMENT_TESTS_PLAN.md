# 用户管理模块测试计划

## 概述
本文档规划了为体育彩票扫盘系统中的用户管理模块编写的测试用例。用户管理模块包含5个主要子页面，提供对系统各类用户的集中管理与操作功能。

## 用户管理模块结构
用户管理模块包含以下5个子页面：
1. **用户管理主页面** - 提供用户列表展示和基础操作入口
2. **用户列表页面** - 显示所有用户详细信息及操作功能
3. **角色权限页面** - 管理用户角色和权限分配
4. **操作日志页面** - 记录用户操作历史
5. **部门管理页面** - 管理用户所属部门结构

## 测试场景

### 真实用户操作场景
1. **日常监控场景**：管理员登录系统，查看用户列表，筛选特定状态的用户
2. **用户管理场景**：管理员添加新用户，设置用户角色和权限
3. **权限分配场景**：管理员修改用户角色，调整权限级别
4. **审核场景**：管理员查看操作日志，审核用户行为
5. **组织架构场景**：管理员设置部门结构，分配用户到不同部门

### 测试数据结构
- **用户数据**：ID、用户名、邮箱、角色、状态、创建时间、更新时间
- **角色权限**：角色ID、角色名称、权限列表、描述
- **操作日志**：日志ID、操作者、操作类型、资源类型、资源ID、操作时间
- **部门数据**：部门ID、部门名称、上级部门、负责人、创建时间

## API端点测试
- `GET /api/v1/admin/backend-users` - 获取后台用户列表
- `POST /api/v1/admin/backend-users` - 创建后台用户
- `PUT /api/v1/admin/backend-users/{id}` - 更新后台用户
- `PATCH /api/v1/admin/backend-users/{id}/status` - 更新用户状态
- `DELETE /api/v1/admin/backend-users/{id}` - 删除后台用户

## 过滤和分页参数
- `page`: 页码
- `size`: 每页数量
- `search`: 搜索关键词（用户名/邮箱）
- `role`: 角色筛选
- `status`: 状态筛选
- `department`: 部门筛选
