# 专抓1-1 开发计划（审批版）

## 1. 目标与范围
**目标**：新增“专抓1-1”功能，对当天竞彩比赛进行 Poisson 1-1 扫盘，给出推荐排名，并提供模型数据详情。
**范围**：
- 前端新增“专抓1-1”页面与“模型数据”详情入口
- 后端新增 1-1 Poisson 模型与当日比赛扫盘服务
- 新增数据抓取任务（自动 + 手动触发）
- 后台菜单：平局预测管理 → 新增“专抓1-1”子菜单

## 2. 需求拆解（与用户条目对齐）
1) **当天竞彩 1-1 扫盘页面**：展示当日比赛列表、1-1 概率、推荐排名、筛选与分页。
2) **Poisson 1-1 模型**：基于大小球盘口与亚洲让球推导 \(\mu_T\) 与 \(\mu_D\)，计算 1-1 概率。
3) **数据抓取**：按照文档建议站点（OddsPortal / Understat / FBref 等）设计抓取流程，支持自动与手动触发。
4) **“模型数据”按钮**：在比赛列表行级按钮，进入详情页/弹窗展示模型输入与中间量。
5) **1-1 推荐排名**：按 1-1 概率或综合评分降序输出。
6) **菜单新增**：平局预测管理 → 专抓1-1。
7) **先计划后开发**：本计划文档审批后进入实现阶段。

## 3. 数据源与采集策略
**模型所需数据类型**：
- 必须：大小球盘口（用于 \(\mu_T\)）、亚洲让球（用于 \(\mu_D\)）
- 推荐：欧赔（校验市场）
- 进阶：xG / 进球数据（模型升级）

**采集来源优先级**（按最新清单）：
- 核心：OddsPortal（大小球/让球/欧赔/历史盘口）
- 进阶增强：Understat（xG），FBref（场均进球/失球、xG/xGA、主客拆分）
- 实时备选：SofaScore / Flashscore / WhoScored
- 量化进阶（可选）：StatsBomb Open Data、BeSoccer（大样本回测）
- 备选（国内）：500w（已用于 `http://localhost:3000/admin/match-data/schedule/jczq` 的竞彩场次抓取）

**使用优先级建议**：
- 新手/实战：OddsPortal + Understat
- 进阶：再加 FBref
- 量化：再加 StatsBomb / 自建数据库

**采集模式**：
- 自动：定时任务（每天固定时间或按开售时间段）
- 手动：后台按钮触发抓取

> 注意：数据质量优先于公式复杂度；第三方站点可能有反爬限制与合规要求，需确认访问策略与许可范围。

## 4. 模型方案（1-1 Poisson，强化版）
**核心思路**：
1) 从大小球盘口估计总进球 \(\mu_T\)
2) 从亚洲让球估计强弱差 \(\mu_D\)
3) 推导 \(\lambda_H, \lambda_A\)：
   \(\lambda_H = (\mu_T + \mu_D)/2\)，\(\lambda_A = (\mu_T - \mu_D)/2\)
4) **收缩（Shrinkage）**：在“均势+小球”场景把 \(\lambda\) 向 1.05 轻度收缩
5) **相关性修正（Dixon–Coles 简化）**：对 1-1 增益 \((1+\rho)\)
6) 最终：
   \(P(1-1)=\big(\lambda'_H e^{-\lambda'_H}\big)\big(\lambda'_A e^{-\lambda'_A}\big)\cdot(1+\rho)\)

**收缩系数建议**：
- AH=0 或 ±0.25 且 O/U ≤2.5 → \(s=0.20\)
- AH=±0.5 或 O/U=2.75 → \(s=0.10\)
- 其他 → \(s=0\)

**相关性修正建议**：
- O/U ≤2.25 且 AH∈{0,±0.25} → \(\rho=+0.10\)
- O/U=2.5 且 AH接近0 → \(\rho=+0.06\)
- 其他 → \(\rho=0\)

**模型输入字段**：
- 大小球盘口（主盘口）
- 亚洲让球盘口（主盘口）
- 赛事信息（联赛、时间、主客队）
- （可选）欧赔/1-1比分赔率（用于价值判断）
- （可选）xG、历史进球均值

**表/模型（建议）**：
- `poisson_11_sources`：抓取原始数据
- `poisson_11_features`：模型输入与中间量（\(\mu_T\), \(\mu_D\), \(\mu_H\), \(\mu_A\)）
