# IP 池自动拉取与运维体验开发计划（草案）

## 背景
- 目标：运维人员可在 `/admin/data-source/ip-pool` 一键开启“自动获取+验证”代理 IP，并获得可观测、可控、可告警的体验。
- 现状：前端使用假数据；后端接口字段有限、验证/调度逻辑占位；与爬虫核心 `proxy_pool`、系统配置及通知未打通。

## 需求要点（运维视角）
- 启停与调度：一键开关“自动拉取+验证”，显示下一次执行时间，支持手动立即执行。
- 质量指标：展示延迟、成功率、最近验证时间、来源、匿名级别、得分/封禁状态；列表可筛选排序；健康度仪表盘与趋势。
- 来源与规则：可配置抓取来源、去重/白名单/黑名单、源级并发与速率限制、最小可用量阈值。
- 验证流水：查看正在验证队列与进度；单个 IP 的验证历史、失败原因、最近 5 次延迟。
- 批量操作：导入/导出（CSV/JSON），批量禁用/删除/重新验证，手动添加并立即验证。
- 通知与告警：可用 IP 低于阈值、连续高失败率、来源超时等，通过钉钉/企微/邮件通知。
- 联动与安全：爬虫任务选用“最佳可用”策略，任务级代理统计；操作审计与敏感字段遮盖。

## 开发计划（预计 2~3 周）
**第 1 周：打通基础链路**
- 修复前端编码与文案；改用后端适配器接口渲染列表/分页/CRUD；添加启停开关与“立即验证/立即拉取”按钮。
- 后端新增字段：`latency_ms`、`success_rate`、`last_checked`、`source`、`anonymity`、`score`、`banned_until`、`fail_reason`，补 schema 与迁移。
- 将 `proxy_pool` 健康检查/评分结果写回 DB；完善 REST：批量验证、导出、批量状态更新。

**第 2 周：体验与监控**
- 前端列表支持多维筛选排序、状态色标、进度条；增加“验证历史”抽屉与“来源/状态”统计卡片。
- 设置页接通 `crawler_proxy_enabled`、拉取频率、验证间隔；展示下一次调度时间并可手动触发。
- 接入日志/事件流（轮询可先行）展示最近验证结果；添加告警阈值配置并打通通知 Webhook。

**第 3 周：可靠性与运维工具**
- 实现代理来源管理（可选源、速率限制、并发上限、黑/白名单）；自动去重与过期清理任务。
- 导入/导出 CSV；批量操作后台任务化，提供进度查询接口。
- 权限与审计：记录操作日志，界面隐藏敏感字段；只读/运维权限分级。
- 补充测试：接口单测（创建/验证/统计/导出）、前端组件测试、调度任务集成测试。

## 立即下一步
1) 确认前端调用 `/admin/ip-pools` 适配器接口；列出需要新增/映射的字段。  
2) 确定“自动拉取”来源列表及调用方式（复用现有脚本或新增采集器）。  
3) 选定通知渠道与阈值默认值，为告警实现做准备。
