三维精算筛选器 - 统一定义手册 
🎯 核心设计目标 
实现基于实力等级差(ΔP)、赢盘等级差(ΔWP)、一赔稳定性(P-Tier)的三维联动筛选系统，支持多选组合查询，快速定位符合特定精算特征的赛事。

📈 维度一：实力等级差 (ΔP) 
定义 
主队与客队的基础实力差值，反映物理层面的强弱对比。
算法 
ΔP = 主队实力值 - 客队实力值
ΔP区间	等级	UI文案	含义
> +25​	+3​	降维打击	主队具备两球及以上优势
+17 ∼ +25​	+2​	强力压制	正路输出非常稳定
+9 ∼ +16​	+1​	主队微优	赢球仍需临场发力
-8 ∼ +8​	0​	旗鼓相当	平局最高发区
-9 ∼ -16​	-1​	客队微优	赢球仍需临场发力
-17 ∼ -25​	-2​	客队压制	主队守和即是胜利
< -25​	-3​	客队统治	主队恐遭遇惨败
筛选值 
[-3, -2, -1, 0, +1, +2, +3]

💰 维度二：赢盘等级差 (ΔWP) 
定义 
基于单队赢盘能力评分(WP)的对撞结果，反映盘路表现和"性格"特征。
计算流程 
Step 1: 单队WP评分 → 分值转换
   WP>1.40 → S(4)
   1.20-1.40 → A(3) 
   0.80-1.20 → B(2)
   0.60-0.80 → C(1)
   WP<0.60 → D(0)

Step 2: ΔWP = 主队分值 - 客队分值
   范围: [-4, +4]整数
ΔWP值	等级	描述
+3, +4​	主极致火热​	主队处于红单巅峰
+1, +2​	主获利走强​	主队赢盘势头正佳
0​	数据均衡​	双方兑现力持平
-1, -2​	客获利走强​	客队守盘效率更优
-3, -4​	客极致火热​	客队盘路兑现力封顶
筛选值 
[-4, -3, -2, -1, 0, +1, +2, +3, +4]

🎲 维度三：一赔稳定性 (P-Tier) 
定义 
基于一赔指数出现频率计算的稳定性等级，反映正路可信度。
算法逻辑 
1. 判断场次类型:   - 双一赔: 主客均有一赔   - 单一赔: 仅一方有一赔     - 无一赔: 双方均无一赔
2. 计算P值:
   双一赔: P = 主一赔% + 客一赔%
   单一赔: P = 唯一一赔% - 对手最高频%
   无一赔: 进入P7，按Ssum = 主最频% + 客最频% 排序
等级判定 
等级	条件	P值范围	描述
S(P1)​	双一赔	P ≥ 140	正路稳胆
A(P2)​	双一赔	110 ≤ P < 140	正路首选
B(P3)​	单一赔	P ≥ 40	正路保障
B-(P4)​	双一赔	P < 110	正路分歧
C(P5)​	单一赔	15 ≤ P < 40	正路存疑
D(P6)​	单一赔	P < 15	正路脆弱
E(P7)​	无一赔或P<0	-	正路缺失
筛选值 
[S, A, B, B-, C, D, E]
对应: [P1, P2, P3, P4, P5, P6, P7]

⚙️ 核心规则与联动逻辑 
1. 排序规则 (强制) 
1. P3场次必须排在P4之前2. 同P级按ΔWP降序排列3. P7场次按Ssum升序排列
2. 降级规则 
当出现以下情况时，P级强制下移一级：

实力等级差方向 与 赢盘等级差方向 背离

(如：实力+3主优，但赢盘-3客热)


实力优势方评级为 D级​ (ΔP为-3的客队统治方)

3. 组合含义示例 
组合	实力ΔP	赢盘ΔWP	稳定性	解读
强势正路​	[+2, +3]	[+3, +4]	[S, A, B]	主队全面占优，正路稳
冷门潜质​	[-1, 0]	[-3, -4]	[D, E]	客队盘路热，主队不稳
均衡博弈​	[0]	[0]	[B, C]	完全均势，临场关键
方向背离​	[+2]	[-2]	[A]	实力主优但盘路客热，降级预警

🔧 技术实现要点 
数据结构 
const filters = {
  strength: ['+1', '+2'],      // 实力等级差
  winLevel: ['3', '4'],        // 赢盘等级差  
  stability: ['S', 'A']        // 稳定性等级
};
筛选逻辑 
// 1. 基础筛选
matches.filter(match => 
  filters.strength.includes(match.strength) &&
  filters.winLevel.includes(match.winLevel) && 
  filters.stability.includes(match.stability)
);
// 2. 规则应用// - 检查P3/P4排序// - 检查方向背离// - 应用降级规则// - 按规则排序

🎯 使用场景示例 
场景1：寻找高概率正路 
实力ΔP: [+2, +3]    (主队显著优势)
赢盘ΔWP: [+3, +4]   (主队盘路火热)  
稳定性: [S, A]      (高稳定性)
→ 主队全面占优，正路概率极高
场景2：挖掘冷门机会 
实力ΔP: [0, -1]     (均势或客微优)
赢盘ΔWP: [-3, -4]   (客队盘路极热)
稳定性: [D, E]      (正路脆弱)
→ 客队可能逆袭，关注冷门
场景3：规避风险场次 
排除: ΔP[0] + ΔWP[0] + 稳定性[E]
→ 避免完全混沌的场次
