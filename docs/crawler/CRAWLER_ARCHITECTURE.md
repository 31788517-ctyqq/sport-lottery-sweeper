## 任务调度架构

### Celery任务流程

```
┌────────────────────────────────────────────────────────────────┐
│                     Celery Beat (定时调度器)                     │
├────────────────────────────────────────────────────────────────┤
│  ⏰ 每30分钟: crawl_all_leagues_concurrent                      │
│  ⏰ 每5分钟:  crawl_upcoming_matches                            │
│  ⏰ 每1小时:  health_check_sources                              │
└──────────────────────┬─────────────────────────────────────────┘
                       │
                       ▼
┌────────────────────────────────────────────────────────────────┐
│                  Celery Worker (任务执行器)                      │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Task: crawl_all_leagues_concurrent(days=3)                    │
│    │                                                            │
│    ├─→ 获取所有活跃联赛                                          │
│    │                                                            │
│    ├─→ 创建并发任务列表                                          │
│    │   [Task1: 英超, Task2: 西甲, Task3: 德甲, ...]            │
│    │                                                            │
│    ├─→ asyncio.gather(*tasks) 【并发执行】                      │
│    │      │                                                     │
│    │      ├─→ Task1 执行中... (3秒)                             │
│    │      ├─→ Task2 执行中... (2.5秒)                           │
│    │      ├─→ Task3 执行中... (2.8秒)                           │
│    │      └─→ ...                                               │
│    │                                                            │
│    ├─→ 等待所有任务完成 (最慢的用时: ~3秒)                       │
│    │                                                            │
│    ├─→ 汇总结果                                                 │
│    │   - 成功: 180场比赛                                         │
│    │   - 失败: 2场比赛                                           │
│    │                                                            │
│    └─→ 保存到数据库                                             │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 并发 vs 串行对比

**串行执行（旧版）**:
```
League1 ──→ League2 ──→ League3 ──→ ... ──→ League20
  3s        3s          3s              3s
                                        
总耗时: 20 × 3s = 60秒
```

**并发执行（新版）**:
```
League1 ──┐
League2 ──┤
League3 ──┤
  ...     ├──→ asyncio.gather() ──→ 完成
League18 ─┤
League19 ─┤
League20 ─┘
  
总耗时: max(所有任务) ≈ 3秒
提升: 20倍
```

```
环境变量 (.env)
   │
   ├─→ SCRAPER_MAX_CONNECTIONS → ScraperEngine.max_connections
   ├─→ SCRAPER_TIMEOUT → ScraperEngine.timeout
   ├─→ SCRAPER_MAX_RETRIES → ScraperEngine.max_retries
   ├─→ SCRAPER_RATE_LIMIT → ScraperEngine.rate_limiter
   ├─→ SCRAPER_ENABLE_CACHE → ScraperEngine.enable_cache
   ├─→ SCRAPER_CACHE_TTL → ScraperEngine.cache_ttl
   └─→ SCRAPER_PROXY_POOL → ScraperEngine.proxy_pool

默认值:
   max_connections = 100
   timeout = 15秒
   max_retries = 3
   rate_limit = 10 req/s
   enable_cache = True
   cache_ttl = 300秒 (5分钟)
```

   │                                      │
   │                                      ▼

## 性能优化点

### 1. 连接池
```
TCPConnector(
    limit=100,          # 总连接数限制
    limit_per_host=30,  # 单主机连接数
    ttl_dns_cache=300   # DNS缓存5分钟
)
```

### 2. 并发控制
```python
# 使用信号量限制并发
semaphore = asyncio.Semaphore(10)

async with semaphore:
    await fetch(url)
```

### 3. 缓存策略
```
L1 缓存: 内存缓存 (5分钟)
   └─→ 命中率目标: 60%+
   └─→ 响应时间: < 0.1秒

L2 缓存: Redis (可选, 1小时)
   └─→ 跨进程共享
   └─→ 持久化
```

### 4. 请求合并
```
多个相同请求 → 只发起一次 → 结果共享
   Request1 ─┐
   Request2 ─┼─→ 合并 → 单次请求 → 分发结果
   Request3 ─┘
```

## 监控指标采集

```
ScraperEngine.stats
   │
   ├─→ total_requests (总请求数)
   ├─→ successful_requests (成功数)
   ├─→ failed_requests (失败数)
   ├─→ cache_hits (缓存命中)
   ├─→ retries (重试次数)
   │
   └─→ 计算指标
       ├─→ success_rate = successful / total * 100%
       └─→ cache_hit_rate = cache_hits / total * 100%

导出到:
   - Prometheus
   - Grafana
   - 日志系统
```

## 总结

这个新架构的核心优势:

✅ **高性能**: 并发+缓存+连接池  
✅ **高可用**: 多层回退+健康检查  
✅ **易扩展**: 统一接口+插件化  
✅ **易维护**: 模块化+文档完善  
✅ **生产就绪**: 重试+限流+监控
