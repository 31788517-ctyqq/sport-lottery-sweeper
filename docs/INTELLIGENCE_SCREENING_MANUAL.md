# 情报系统筛选管理手册

## 1. 系统概述

情报系统是体育彩票扫盘系统的核心分析模块，主要用于对海量体育赛事数据进行智能筛选和分析，帮助用户发现潜在的投注机会和价值信息。系统通过定义筛选规则、执行筛选任务、分析筛选结果三个主要环节，实现对数据的深度挖掘和价值提炼。

### 1.1 主要功能

1. **数据筛选**：通过多种筛选方式（条件筛选、正则匹配等）从原始数据中提取有价值信息
2. **任务管理**：管理筛选任务的执行和监控
3. **结果分析**：对筛选结果进行统计分析和可视化展示
4. **规则管理**：创建、编辑和管理筛选规则库
5. **质量评估**：评估筛选结果的质量和准确性

## 2. 筛选管理页面详解

访问路径：`http://localhost:3000/admin/intelligence/screening`

该页面是情报系统的核心操作界面，采用标签页设计，分为四个主要功能区域：

### 2.1 筛选规则管理

#### 页面布局
- **统计卡片区**：显示活跃规则数量等关键指标
- **搜索过滤区**：提供规则名称、状态和时间范围筛选
- **规则列表**：以表格形式展示所有筛选规则

#### 用户操作及目的

| 操作 | 操作方法 | 操作目的 |
|------|----------|----------|
| 新建规则 | 点击"新建筛选规则"按钮 | 创建新的数据筛选条件，如"主胜赔率<2.0"或"进球数>2.5" |
| 编辑规则 | 在规则列表中点击"编辑"按钮 | 修改已有规则的配置，优化筛选条件 |
| 执行规则 | 在规则列表中点击"执行"按钮 | 立即应用该规则筛选最新数据，获取实时结果 |
| 删除规则 | 在规则列表中点击"删除"按钮 | 移除不再需要的筛选规则，保持规则库的精简 |
| 规则查询 | 在搜索区填写条件后点击"查询" | 快速定位特定规则，提高管理效率 |

**规则配置说明**：
- **规则名称**：规则的标识，应描述规则功能（如"高价值比赛筛选"）
- **筛选类型**：
  - 条件筛选：基于数值比较的简单条件
  - 模糊匹配：基于相似度的匹配
  - 正则匹配：使用正则表达式进行复杂匹配
  - 机器学习：基于AI模型的智能筛选
- **规则配置**：JSON格式的筛选条件定义，如：
  ```json
  {"odds": {"home_win": {"min": 1.5, "max": 2.0}}, "goal_expectation": ">2.5"}
  ```

### 2.2 筛选任务管理

#### 页面布局
- **任务统计区**：显示今日任务、成功任务等关键指标
- **任务时间线**：以时间轴形式展示最近执行的任务

#### 用户操作及目的

| 操作 | 操作方法 | 操作目的 |
|------|----------|----------|
| 批量筛选 | 点击"批量筛选"按钮 | 同时执行多个筛选规则，提高数据处理效率 |
| 查看任务详情 | 点击时间线中的任务条目 | 了解任务执行详情，包括状态、描述等信息 |

**任务管理价值**：
- 监控系统运行状态
- 识别执行失败的任务并进行问题排查
- 评估任务执行效率和成功率

### 2.3 结果分析

#### 页面布局
- **图表区**：展示筛选结果分布和规则效果对比
- **热门结果表**：列出匹配次数最多、准确率最高的筛选结果

#### 用户操作及目的

| 操作 | 操作方法 | 操作目的 |
|------|----------|----------|
| 查看结果分布 | 查看"筛选结果分布"图表 | 了解不同筛选结果的占比情况 |
| 比较规则效果 | 查看"规则效果对比"图表 | 评估不同规则的筛选效果和准确性 |
| 分析热门结果 | 查看"热门筛选结果"表格 | 识别最有价值的筛选条件和模式 |

**分析价值**：
- 评估筛选规则的有效性
- 发现数据中的隐藏模式和规律
- 为优化筛选规则提供数据支持

### 2.4 规则模板管理

#### 页面布局
- **模板网格**：以卡片形式展示可用的规则模板

#### 用户操作及目的

| 操作 | 操作方法 | 操作目的 |
|------|----------|----------|
| 使用模板 | 点击模板卡片 | 快速应用经过验证的有效筛选规则 |
| 查看模板详情 | 点击模板卡片 | 了解模板的详细配置和使用场景 |

**模板价值**：
- 提供经过验证的有效筛选条件
- 减少用户配置规则的学习成本
- 提高筛选效率和准确性

## 3. 系统工作流程

1. **规则创建**：用户定义筛选条件（如"英超联赛+主胜赔率<2.0"）
2. **任务执行**：系统应用规则筛选最新赛事数据
3. **结果生成**：系统返回匹配的赛事列表及其相关数据
4. **结果分析**：系统统计筛选结果的分布和质量
5. **决策支持**：用户根据筛选结果做出投注决策

## 4. 高级功能说明

### 4.1 批量筛选
- **操作方法**：点击"批量筛选"按钮
- **目的**：同时执行多个筛选规则，全面覆盖不同投注机会
- **应用场景**：赛前全面分析，寻找所有潜在价值投注点

### 4.2 规则质量评估
- **操作方法**：查看结果分析中的准确率指标
- **目的**：评估筛选规则的有效性
- **应用场景**：优化筛选规则，提高筛选结果的准确性

### 4.3 数据导出
- **操作方法**：点击"导出结果"按钮
- **目的**：将筛选结果导出为CSV或JSON格式
- **应用场景**：进一步分析或与其他系统集成

## 5. 最佳实践建议

1. **规则命名规范**：使用描述性名称（如"英超高价值比赛筛选"）
2. **规则测试流程**：
   - 先在小数据集上测试新规则
   - 评估准确率后再应用于全量数据
3. **模板应用策略**：
   - 从高成功率模板开始
   - 根据实际需求调整模板参数
4. **结果验证方法**：
   - 对比筛选结果与实际赛果
   - 定期评估规则的有效性

## 6. API接口说明

### 6.1 情报数据API
后端API位于 [backend/api/v1/intelligence.py](../backend/api/v1/intelligence.py)，提供以下功能：

- `GET /intelligence/` - 获取情报列表
- `GET /intelligence/recent` - 获取最近的情报
- `GET /intelligence/high-priority` - 获取高优先级情报
- `POST /intelligence/` - 创建情报
- `PUT /intelligence/{id}` - 更新情报
- `DELETE /intelligence/{id}` - 删除情报
- `GET /intelligence/types/` - 获取情报类型列表
- `GET /intelligence/sources/` - 获取信息来源列表
- `GET /intelligence/match/{match_id}` - 获取比赛相关情报
- `POST /intelligence/filter` - 高级筛选情报
- `GET /intelligence/stats/summary` - 获取情报统计摘要

### 6.2 情报服务
后端服务位于 [backend/services/intelligence_service.py](../backend/services/intelligence_service.py)，提供以下功能：

- 统计信息获取
- 趋势分析
- 标记无效情报
- 重新抓取数据
- 批量标记数据
- 数据导出功能
- 错误分布分析
- 情报质量分析

情报系统通过科学的数据筛选和分析，帮助用户从海量体育赛事数据中快速定位价值投注机会，显著提高投注决策的科学性和准确性。建议用户定期优化筛选规则，关注结果分析指标，不断提升系统使用效果。
