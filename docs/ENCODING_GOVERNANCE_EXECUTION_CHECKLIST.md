## 4. 验收标准（统一量化）

1. 编码合规：文本文件 UTF-8 合规率 100%（白名单除外）。
2. 乱码扫描：命中关键乱码特征词为 0（如 `鏁版`、`鐢ㄦ埛`、`锟`、`�`）。
3. 构建与语法：前端构建通过，后端关键模块编译通过。
4. 功能冒烟：登录、菜单、数据源管理、用户管理、官方信息管理全通过。
5. 门禁验证：本地钩子与 CI 都能拦截故意注入乱码样本。

## 6. 风险与控制

1. 风险：批量转码导致二次损坏。  
控制：禁止“一键全仓转码”，按目录分批修复并复核。

2. 风险：不可逆乱码无法自动恢复。  
控制：优先从 Git 历史恢复，无法恢复再人工校对。

3. 风险：多 IDE 再次引入乱码。  
控制：提交门禁 + CI 必过 + 分支保护。
