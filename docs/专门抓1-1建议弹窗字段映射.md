# Poisson-11“建议”弹窗字段-现有数据字段映射文案

## 1. 适用范围
本映射用于页面：`/admin/draw-prediction/poisson-11` 的“建议”弹窗。
目标：把“弹窗展示字段”与“当前已存在数据字段”一一对应，明确取值规则与缺失兜底（统一显示 `-`）。

## 2. 当前可用数据入口

### 2.1 列表接口
- `GET /api/v1/draw-prediction/poisson-11/list`
- 主要字段：`match_id/number/match_time/league/home_team/away_team/mu_total/mu_diff/mu_home/mu_away/prob_11/rank/data_source`

### 2.2 详情接口
- `GET /api/v1/draw-prediction/poisson-11/detail/{match_id}`
- 主要字段：
  - 顶层：`match_id/match_date/match_time/league/home_team/away_team/number/mu_total/mu_diff/mu_home/mu_away/prob_11/rank/data_source`
  - `input_payload`：
    - `total_goals_line`
    - `total_goals_line_source`
    - `handicap`
    - `handicap_source`
    - `odds.win / odds.draw / odds.lose`
    - `mu.total / mu.diff / mu.home / mu.away`
    - `quality_flags[]`
    - `source_attributes{...}`（源数据透传）

### 2.3 source_attributes 中常见可用字段（当前）
- 基础：`source_schedule_date/status_des/full_score(or score)/halftime_score(or half_score)/handicap_0/handicap/source_url`
- SP：`odds_win/odds_draw/odds_lose/odds_nspf_*/odds_spf_*`
- 其它赔率标签：`other_odds_tabs.eu / other_odds_tabs.asia / other_odds_tabs.goals`

### 2.4 1-1比分赔率（500 北单比分玩法）数据口径
- 页面源：`https://trade.500.com/bjdc/project_fq_bf.php?expect={period}`
- 期号字段：`#expect`、`#expect_select`、`responseJson.period`
- 场次键：`tr.vs_lines` 的 `value` 属性（含 `index/scheduleDate/endTime/homeTeam/guestTeam`）+ `<span class="chnum">`
- 赔率节点：对应 `tr#sp_lines_{index}` 中，定位 `span.f_bf` 文本为 `1:1` 的 label，读取同 label 内 `span.sp_value`
- 页面空值：若显示 `--` 或空串，则判定为无赔率
- XML回退：`https://www.500.com/static/public/bjdc/xml/sp/just_{period}_42.xml`
  - `playid=42` 为“比分”
  - `1:1` 在 codeValue 中为第13个选项，对应 XML 属性 `c13`
  - `c13` 为空、`0`、非数值时，判定为无赔率

## 4. 当前“已就绪可落地”字段清单（MVP）
建议第一版弹窗先落这些已就绪字段：
1. 比赛信息：编号、赛事、对阵、开赛时间。
2. 模型信息：`P(1-1)、rank、μT、μD、μH、μA`。
3. 结构信息：`大小球区间、让球区间、平赔区间、结构标签`。
4. 风险信息（基础版）：`数据缺失风险、盘口异常风险、状态/比分提示`。

说明：`1-1赔率` 来源已明确为 500 比分页（含 XML 回退）；在抓取任务上线前，`1-1赔率/P_imp/Edge` 仍展示为 `-`。

## 5. 字段缺失统一规范
1. 任意字段取值为空、`null`、`undefined`、非数值异常时，统一显示 `-`。
2. 依赖上游字段的计算项（如 `P_imp`、`Edge`）若依赖缺失，结果显示 `-`。
3. 文案型结论若核心字段缺失，统一输出：`数据不足，建议观望`。

## 6. 建议弹窗内部计算口径（文案版）

### 6.1 模型强度分（40分）
- 依据：`prob_11`、`rank`、`μT/μD` 结构合理性。

### 6.2 价值优势分（30分）
- 依据：`Odds_11` 与 `P_model` 的偏差（数据源已明确；未抓到赔率时该子项置灰）。

### 6.3 结构匹配分（20分）
- 依据：`O/U`、`AH`、`平赔` 是否落在“1-1甜点区”。

### 6.4 风险控制分（10分）
- 依据：`quality_flags`、盘口波动、状态异常等。
