# 平局预测管理验收测试用例（闭环 + 模拟数据）

## 1. 测试目标
- 验证四个页面功能串联可用。
- 验证每页至少1条可见数据。
- 验证 `data-features` 现有17条特征被保留。

## 2. 测试环境前置条件
1. 管理后台可登录。
2. 四个页面可访问。
3. 已执行模拟数据初始化（或手工录入）：
   - 训练任务 >= 1
   - 模型版本 >= 1
   - 预测结果 >= 1
4. `data-features` 初始数量为17。

## 3. 测试用例总览

| 用例ID | 模块 | 用例名称 | 优先级 |
|---|---|---|---|
| DP-001 | Data Features | 特征数量与保留校验 | P0 |
| DP-002 | Training Evaluation | 训练任务首屏可见 | P0 |
| DP-003 | Model Deployment | 模型版本首屏可见 | P0 |
| DP-004 | Prediction Monitoring | 预测记录首屏可见 | P0 |
| DP-005 | 闭环链路 | 四页完整流转 | P0 |
| DP-006 | 追溯性 | 预测可反查模型与任务 | P1 |
| DP-007 | 空态治理 | 上游为空时的引导行为 | P1 |
| DP-008 | 状态一致性 | 页面状态字典一致 | P1 |

## 4. 详细测试用例

## 4.1 DP-001 特征数量与保留校验
- 前置条件：`data-features` 已有17条历史特征。
- 步骤：
  1. 打开 `/admin/draw-prediction/data-features`。
  2. 记录列表总数。
  3. 切换刷新/分页后再次记录总数。
- 预期结果：
  1. 总数始终为17。
  2. 无数据丢失、无被覆盖。

## 4.2 DP-002 训练任务首屏可见
- 步骤：
  1. 打开 `/admin/draw-prediction/training-evaluation`。
  2. 查看列表首屏。
- 预期结果：
  1. 至少有1条任务记录。
  2. 任务含状态与关键指标字段。

## 4.3 DP-003 模型版本首屏可见
- 步骤：
  1. 打开 `/admin/draw-prediction/model-deployment`。
  2. 查看列表首屏。
- 预期结果：
  1. 至少有1条模型版本记录。
  2. 记录包含 `version_tag`、`training_job_id`、`status`。

## 4.4 DP-004 预测记录首屏可见
- 步骤：
  1. 打开 `/admin/draw-prediction/prediction-monitoring`。
  2. 查看统计卡片与列表。
- 预期结果：
  1. 至少有1条预测记录。
  2. 统计卡片 `total >= 1`。
  3. 列表字段完整（match_id、概率、赛果、时间）。

## 4.5 DP-005 四页完整流转
- 步骤：
  1. 在 `data-features` 选定特征。
  2. 进入 `training-evaluation` 创建训练任务。
  3. 训练成功后进入 `model-deployment` 完成部署。
  4. 进入 `prediction-monitoring` 查看该模型效果。
- 预期结果：
  1. 页面可顺畅跳转。
  2. 不需要人工查库补信息。
  3. 无“流程断点”。

## 4.6 DP-006 预测追溯校验
- 步骤：
  1. 在 `prediction-monitoring` 选一条记录。
  2. 检查是否可定位对应 `model_version_id`。
  3. 再反查 `training_job_id` 与特征集。
- 预期结果：
  1. 链路完整可追溯。
  2. 无孤立预测记录。

## 4.7 DP-007 空态治理
- 步骤：
  1. 构造模型为空场景（测试环境）。
  2. 打开 `model-deployment`。
- 预期结果：
  1. 展示明确引导：“先去训练评估创建任务”。
  2. 提供可点击跳转按钮。

## 4.8 DP-008 状态一致性
- 步骤：
  1. 对照四页状态文案与接口返回状态值。
  2. 检查同一状态在不同页面是否同义显示。
- 预期结果：
  1. `pending/running/success/failed` 不混乱。
  2. `active/inactive` 与部署文案一致。

## 5. 模拟数据专项验收

| 校验项 | 预期值 |
|---|---|
| data-features 数量 | 17（保留不变） |
| training-evaluation | >=1 条模拟任务 |
| model-deployment | >=1 条模拟模型 |
| prediction-monitoring | >=1 条模拟预测 |

## 6. 通过标准（Go/No-Go）
- Go：
  - P0 用例全部通过。
  - 模拟数据专项四项全部达标。
  - 闭环链路可完整演示。
- No-Go：
  - 任一页面仍为空。
  - `data-features` 非17条。
  - 预测无法追溯至模型和训练任务。
