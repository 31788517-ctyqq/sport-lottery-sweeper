# 爬虫源配置管理功能设计说明

## 1. 功能定位
统一管理爬虫数据源的参数与策略，支持**全局参数**与**单源差异化配置**，并提供配置版本管理、批量导入导出能力，确保爬虫系统灵活可维护。

## 2. 功能范围

| 模块 | 功能点 | 说明 |
|------|--------|------|
| **全局参数配置** | 默认请求头、超时、重试次数、代理池策略 | 适用于所有爬虫源的默认抓取策略 |
|  | 公共 User-Agent 列表与轮换规则 | 防止单一 UA 被目标网站封禁 |
| **单源参数配置** | 抓取频率、深度、页面解析规则 | 针对不同站点的定制化抓取策略 |
|  | 自定义请求参数与认证信息（Cookie、Token） | 支持需要登录或特殊鉴权的站点 |
| **配置版本管理** | 保存历史配置快照 | 每次保存生成版本号 |
|  | 一键回滚至历史版本 | 快速恢复误改前的配置 |
| **导入/导出** | Python/YAML/JSON 格式批量导入 | 跨环境迁移或批量初始化 |
|  | 导出当前配置为 YAML/JSON | 备份与审计 |
| **界面交互** | 左侧树形结构列出所有数据源 | 全局/单源分层展示 |
|  | 右侧表单编辑参数，JSON 高亮显示 | 直观编辑复杂参数 |
|  | 顶部操作栏：新增、复制、删除、保存、测试连接、导入、导出、回滚 | 一站式操作 |

## 3. 界面原型说明
- **左侧**：树形组件展示数据源层级（全局参数 + 各站点配置节点）  
- **右侧**：JSON 文本编辑区（等宽字体、语法高亮）+ 保存按钮  
- **顶部操作栏**：功能按钮组，依据权限动态显示  
- **整体嵌入**：页面嵌入后台管理系统的右侧窗口区域，不单独占满浏览器窗口  

## 4. 使用流程说明

### 4.1 进入页面
1. 登录后台管理系统  
2. 在左侧导航菜单选择 **爬虫管理 → 源配置**  
3. 系统加载 `/admin/crawler/source-config` 页面，右侧窗口显示数据源树与默认配置（全局参数）

### 4.2 选择数据源
1. 在左侧树形结构中点击目标节点（如“Site A”）  
2. 右侧表单自动加载该节点的配置（JSON 格式）  

### 4.3 编辑配置
1. 在右侧 JSON 编辑区修改参数（如 `timeout`、`frequency`、`headers` 等）  
2. 确保 JSON 语法正确（否则保存时会提示错误）  

### 4.4 保存配置
1. 点击 **保存** 按钮  
2. 系统校验 JSON 合法性  
3. 校验通过后调用后端 API 保存为新版本，并记录版本号与时间  
4. 提示“保存成功”，新配置立即生效（或按策略延迟生效）  

### 4.5 版本回滚
1. 点击 **回滚** 按钮  
2. 弹出历史版本列表（版本号、时间、备注）  
3. 选择目标版本，确认回滚  
4. 系统将配置恢复至选定版本，并生成新的版本记录  

### 4.6 导入配置
1. 点击 **导入** 按钮  
2. 选择文件（支持 `.json` / `.yaml` / `.py` 格式）  
3. 系统解析并校验格式与必填字段  
4. 校验通过后合并或替换当前配置，生成新版本  

### 4.7 导出配置
1. 点击 **导出** 按钮  
2. 选择导出格式（JSON / YAML）  
3. 系统生成文件并下载到本地  

### 4.8 测试连接
1. 选中某单源节点  
2. 点击 **测试连接**  
3. 系统使用当前配置发起请求，返回响应状态与耗时  
4. 用于验证认证信息、代理、请求头等是否可用  

### 4.9 新增/复制/删除
- **新增源**：在树形结构右键或点击“新增源”按钮，填写基本信息后生成新节点  
- **复制配置**：选中节点后点击“复制配置”，生成相同配置的新节点，可改名后保存  
- **删除**：选中节点后点击“删除”，需二次确认，防止误删  

## 5. 权限控制（建议）
- **只读角色**：仅查看与导出配置  
- **编辑角色**：允许修改、保存、导入、测试连接  
- **管理员角色**：拥有全部操作权限，包括版本回滚与删除  

## 6. 技术实现要点
- 前端：Vue 3 + Element Plus，树形组件 `el-tree`，JSON 编辑区使用 `el-input type=textarea`（可升级 Monaco Editor）  
- 路由：`/admin/crawler/source-config` 嵌入主布局右侧窗口  
- 状态管理：可按需接入 Pinia 管理配置数据与版本历史  
- 后端接口（示例）：  
  - `GET /api/crawler/source-config/:id` 获取配置  
  - `POST /api/crawler/source-config` 保存新配置（含版本记录）  
  - `GET /api/crawler/source-config/:id/versions` 获取历史版本  
  - `POST /api/crawler/source-config/:id/rollback` 回滚  
  - `POST /api/crawler/source-config/import` 导入  
  - `GET /api/crawler/source-config/export?format=json` 导出  

## 7. 注意事项
- JSON 配置需符合后端解析 schema，避免运行时错误  
- 导入大文件时注意超时与内存限制  
- 版本回滚不可逆，建议在关键回滚前再次导出当前配置备份  
- 测试连接仅验证连通性，不代表长期可用性，生产环境需定期巡检
