# 专门抓1-1赚钱思路（Poisson-11“建议”弹窗文案稿）

## 1. 目标
在 `http://localhost:3000/admin/draw-prediction/poisson-11` 的每场比赛“操作”列新增“建议”弹窗后，弹窗应承担三件事：
1. 统一判断是否投注（可投/观察/放弃）。
2. 把模型结论、赔率价值、风险项放在同一页里，避免只看单一指标。
3. 给出明确的仓位建议与放弃理由，便于复盘。

## 2. 弹窗定位文案（标题区）
- 主标题：`1-1投注建议`
- 副标题：`基于Poisson 1-1模型 + 赔率价值 + 风险约束的决策框架`
- 风险提示：`提示：本建议用于筛选“有价值机会”，不保证单场命中。`

## 3. 弹窗结构与文案

### A. 结论卡（第一屏，先看结论）
- 字段：
  - `建议等级`：`强建议 / 可跟进 / 仅观察 / 放弃`
  - `操作建议`：`建议投注1-1 / 小注试投 / 继续观察 / 不投注`
  - `建议仓位`：`0.25u / 0.5u / 1.0u / -`
  - `核心理由`：最多3条，短句化
- 结论文案模板：
  - `建议等级：{{level}}`
  - `操作建议：{{action}}`
  - `建议仓位：{{stake}}（单场不超过总资金{{cap_pct}}）`
  - `核心理由：{{reason_1}}；{{reason_2}}；{{reason_3}}`

### B. 模型面（Poisson强度）
- 字段：
  - `P(1-1)`
  - `排名`
  - `μT（总进球）`
  - `μD（强弱差）`
  - `模型来源完整度`
- 解释文案：
  - `模型倾向：{{model_bias}}`
  - `解读：μT偏低且μD接近0时，1-1概率更具结构优势。`

### C. 价值面（赔率是否值得买）
- 字段：
  - `1-1赔率`（若缺失显示`-`）
  - `隐含概率 P_imp = 1/Odds_11`
  - `模型概率 P_model`
  - `价值差 Edge = P_model - P_imp`
  - `安全垫 Margin`（默认8%）
- 数据来源（已确认）：
  - `主抓`：`https://trade.500.com/bjdc/project_fq_bf.php?expect={period}`
  - `抓取位`：`sp_lines_{index}` 中 `f_bf=1:1` 对应 `sp_value`
  - `回退`：`https://www.500.com/static/public/bjdc/xml/sp/just_{period}_42.xml` 的 `c13`
  - `匹配键`：优先 `period + 场次编号(index/chnum)`，回退“日期+对阵+开赛时间”
- 价值判断文案：
  - `价值结论：{{value_result}}`
  - `判定规则：当 P_model > P_imp × (1 + Margin) 时，视为具备价值。`

### D. 盘口结构面（甜点区匹配）
- 字段：
  - `大小球区间`：`2.0-2.5 / 2.75 / ≥3.0`
  - `亚洲让球区间`：`0 / ±0.25 / ±0.5 / 其他`
  - `平赔区间`：`2.8-3.1 / 其他`
  - `结构标签`：`甜点区 / 次甜点 / 非甜点`
- 解释文案：
  - `结构结论：{{structure_result}}`
  - `解读：越接近“小球+均势+平赔合理”组合，越支持1-1场景。`

### E. 风险面（一票否决）
- 风险项：
  - `阵容风险`（主力伤停/轮换不明）
  - `战意风险`（争冠/保级/淘汰赛必须分胜负）
  - `赛程风险`（密集赛程、明显体能差）
  - `盘口异常`（临场剧烈波动、单边过热）
  - `数据缺失`（关键字段缺失）
- 显示规则：
  - 任一高风险触发时，`建议等级`最多降为`仅观察`。
  - 触发“一票否决”时直接`放弃`。
- 文案模板：
  - `风险结论：{{risk_level}}`
  - `触发项：{{risk_item_list}}`

### F. 资金与执行面（避免情绪单）
- 字段：
  - `单场上限`（建议总资金0.5%-2%）
  - `当日总上限`（建议总资金3%-5%）
  - `连亏保护`（如连续3场未中暂停）
- 文案模板：
  - `资金建议：{{bankroll_action}}`
  - `执行纪律：命中阈值才下单，不因临场情绪加码。`

## 4. 建议等级判定标准（用于弹窗结论）

### 4.1 评分建议（100分制）
- `模型强度`：40分
- `价值优势`：30分
- `盘口结构匹配`：20分
- `风险控制`：10分

### 4.2 分级规则
- `80-100`：`强建议`（可考虑0.75u-1.0u）
- `65-79`：`可跟进`（可考虑0.25u-0.5u）
- `50-64`：`仅观察`（原则上不下单）
- `<50`：`放弃`

### 4.3 一票否决条件
出现以下任一项，直接`放弃`：
1. 主抓 + 回退后 `1-1赔率` 仍缺失，或关键概率无法计算。
2. `Edge <= 0` 且无其他强证据。
3. 临场重大伤停且方向不确定。
4. 明显“必须分胜负”战意场景。

## 5. 弹窗底部固定说明文案
- `说明1：该建议是“概率与价值”框架，不是赛果承诺。`
- `说明2：长期盈利依赖纪律执行，而非单场命中。`
- `说明3：无数据或来源不完整的字段统一显示“-”。`

## 6. 可直接上屏的“结论句”模板
- 强建议：`模型与赔率同时支持，且风险可控，满足出手条件。`
- 可跟进：`有一定价值优势，但存在轻微不确定项，建议小注。`
- 仅观察：`模型信号一般或价值不足，建议观望等待更优机会。`
- 放弃：`不满足价值条件或触发高风险项，本场不投注。`

## 7. 复盘记录建议（给后续优化用）
每场建议弹窗应沉淀以下复盘字段：
- `赛前建议等级`
- `是否实际投注`
- `投注仓位`
- `关键理由`
- `赛果（是否1-1）`
- `复盘结论（判断正确/错误原因）`

> 目标：让“建议弹窗”从一次性提示，升级为可持续迭代的决策系统。
