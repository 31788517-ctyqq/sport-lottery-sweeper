# 智能体模板管理指南

## 概述

智能体模板管理是体育彩票扫盘系统AI服务模块的核心功能之一。它允许管理员创建、配置和管理可重用的智能体模板，用于快速部署各种AI智能体（如赔率监控、推荐、预测、数据分析等）。

## 功能特性

### 1. 模板管理
- **模板列表**：查看所有智能体模板，支持分页和筛选
- **模板详情**：查看模板的详细配置和使用统计
- **创建模板**：通过表单创建新的智能体模板
- **编辑模板**：修改现有模板的配置
- **删除模板**：删除不再需要的模板

### 2. 模板配置
- **基础信息**：模板名称、类型、描述、分类
- **标签系统**：支持多标签分类和筛选
- **难度级别**：简单、中等、困难三级难度
- **配置参数**：
  - 智能体配置（JSON格式）
  - 链配置（JSON格式）
  - 工具配置（JSON格式）
- **发布状态**：控制模板是否对外可用

### 3. 批量操作
- **批量发布**：同时发布多个未发布的模板
- **批量取消发布**：同时取消多个已发布模板
- **批量删除**：同时删除多个模板

### 4. 使用统计
- **使用次数**：模板被使用的总次数
- **成功率**：基于模板创建的智能体平均成功率
- **执行时间**：平均任务执行时间
- **趋势分析**：使用情况变化趋势

## API接口

### 基础信息
- **API路径**：`/api/v1/agents/templates`
- **认证方式**：Bearer Token（需要管理员权限）
- **数据格式**：JSON

### 主要端点

#### 1. 获取模板列表
```
GET /api/v1/agents/templates
查询参数：
- page: 页码（默认1）
- page_size: 每页大小（默认20）
- template_type: 模板类型筛选
- category: 分类筛选
- published: 发布状态筛选
- name: 名称模糊搜索
```

#### 2. 获取模板详情
```
GET /api/v1/agents/templates/{template_id}
```

#### 3. 创建模板
```
POST /api/v1/agents/templates
请求体：AgentTemplateCreate 模型
```

#### 4. 更新模板
```
PUT /api/v1/agents/templates/{template_id}
请求体：AgentTemplateUpdate 模型
```

#### 5. 删除模板
```
DELETE /api/v1/agents/templates/{template_id}
```

#### 6. 发布/取消发布模板
```
POST /api/v1/agents/templates/{template_id}/publish
POST /api/v1/agents/templates/{template_id}/unpublish
```

## 使用流程

### 1. 创建模板
1. 点击"创建模板"按钮
2. 填写模板基本信息：
   - 模板名称（必填）
   - 模板类型（必填）
   - 描述（可选）
   - 分类（可选）
   - 标签（可选）
3. 配置参数：
   - 智能体配置（JSON格式）
   - 链配置（JSON格式）
   - 工具配置（JSON格式）
4. 设置难度级别
5. 选择是否立即发布
6. 点击"确定"保存

### 2. 使用模板创建智能体
1. 在模板列表中找到需要的模板
2. 点击"使用此模板"按钮
3. 系统将基于模板配置创建新的智能体
4. 自动填充智能体的基础配置
5. 可进一步调整特定参数

### 3. 管理模板
1. **编辑模板**：点击"编辑"按钮修改配置
2. **发布/取消发布**：点击发布状态按钮切换
3. **查看详情**：点击"详情"按钮查看完整信息
4. **删除模板**：点击"删除"按钮移除模板

## 权限控制

### 用户角色权限
| 角色 | 查看模板 | 创建模板 | 编辑模板 | 删除模板 | 发布/取消发布 |
|------|----------|----------|----------|----------|---------------|
| 管理员 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 操作员 | ✓ | ✓ | ✓ | ✗ | ✓ |
| 查看者 | ✓ | ✗ | ✗ | ✗ | ✗ |

### 操作限制
1. 只有已发布的模板才能被普通用户使用
2. 模板创建后默认未发布状态
3. 删除模板为软删除，可恢复
4. 系统内置模板不可删除

## 最佳实践

### 1. 模板设计
- **明确用途**：每个模板应有明确的用途和场景
- **适度抽象**：保持一定通用性，但不过度复杂
- **文档完善**：提供清晰的配置说明和使用示例
- **版本控制**：重要更新时创建新版本模板

### 2. 配置管理
- **参数验证**：确保配置参数的完整性和有效性
- **默认值**：为可选参数提供合理的默认值
- **错误处理**：配置明确的错误处理机制
- **性能优化**：避免配置导致的性能问题

### 3. 使用建议
- **模板复用**：优先使用现有模板，避免重复配置
- **渐进式配置**：从简单模板开始，逐步增加复杂性
- **定期审查**：定期检查和更新模板配置
- **性能监控**：监控基于模板创建的智能体性能

## 故障排除

### 常见问题

#### 1. 模板创建失败
- **问题**：提交表单后提示保存失败
- **原因**：配置JSON格式错误或必填字段缺失
- **解决**：检查JSON格式，确保必填字段已填写

#### 2. 模板无法发布
- **问题**：点击发布按钮无响应或报错
- **原因**：权限不足或模板配置不完整
- **解决**：检查用户权限，确保模板配置完整

#### 3. 模板详情无法加载
- **问题**：点击详情按钮无响应或显示错误
- **原因**：模板已被删除或数据库连接问题
- **解决**：检查模板状态，确认数据库正常

#### 4. 批量操作失败
- **问题**：批量操作时部分操作失败
- **原因**：部分模板状态不满足操作条件
- **解决**：检查所有选中模板的状态

### 技术支持
1. **日志查看**：检查后端日志获取详细错误信息
2. **API测试**：使用Postman等工具直接测试API端点
3. **数据库检查**：确认数据库表结构和数据完整性
4. **网络连接**：检查前后端服务通信是否正常
