# 北单三维筛选器前端页面文案与接口说明

## 1. 页面概述
- **页面标题**：北单过滤面板  
- **访问路径**：`/admin/beidan-filter`  
- **功能**：提供策略筛选、条件过滤、排序、结果展示的完整操作界面。  
- 页面分为多个功能卡片，分别处理策略维度、其它条件、排序方式、策略管理与结果展示。

---

## 2. 卡片分组与功能说明

### 2.1 筛选卡片头部（FilterCardHeader）
- **实时匹配 x 场**  
  - 文案示例：`实时匹配 38 场`  
  - 数据来源：后端接口 `GET /api/latest-matches-count`（返回 `{ count, latestDate }`）。  
  - 作用：显示最新 `date_time` 比赛场次数。  
  - 点击“获取实时数据”按钮后更新，并刷新“其它条件卡片”的 `date_time` 下拉选项为最新 3 期。

- **获取实时数据** 按钮  
  - 功能：获取最新比赛数量并刷新 `date_time` 下拉选项。  
  - 不触发策略筛选，仅更新实时数据计数和日期选项。

- **显示P级规则** 按钮  
  - 功能：弹出 P 级规则说明（目前为提示信息）。

---

### 2.2 策略维度卡片（FilterSection 策略维度部分）
> 与策略维度无关的其他条件在下方单独分组

- **实力等级差 ΔP**  
  - 选项：`-3 ~ +3` 共 7 档，每档含数值、区间、描述（如 `-3: < -25 客队实力碾压`）。  
  - 多选，可叠加。  
  - 对应后端参数：`powerDiffs`（映射关系见前端 `mapStrengthToPowerDiffs`）。

- **赢盘等级差 ΔWP**  
  - 选项：`-4 ~ +4` 共 9 档，每档含数值、区间、描述（如 `+4: S 主极致火热`）。  
  - 多选，可叠加。  
  - 对应后端参数：`winPanDiffs`（映射关系见前端 `mapWinPanToWinPanDiffs`）。

- **一赔稳定性 P-Tier**  
  - 选项：`S、A、B、B-、C、D、E` 共 7 档，每档含区间、描述（如 `S: P1 正路稳胆`）。  
  - 多选，可叠加。  
  - 对应后端参数：`stabilityTiers`。

- **示例策略**（强势正路、冷门潜质、均衡博弈）  
  - 文案：仅为示例选择状态，点击后加载对应条件到上方三个维度，**不自动应用筛选**。  
  - 作用：帮助用户快速预览条件组合。  
  - 需手动点击“应用筛选”才会生成“当前策略”并筛选。

- **应用筛选** 按钮  
  - 功能：根据当前三维条件生成“当前策略”临时策略，并立即执行筛选。  
  - 后端接口：`POST /api/filter`（或相应筛选接口），参数见 3.1 表。  
  - 返回：`totalResults`、`pagedResults`。

- **保存当前策略** 按钮  
  - 功能：将“当前策略”保存为可复用的命名策略。  
  - 弹出输入框让用户输入策略名称（1-20字符）。  
  - 后端接口：`POST /api/strategies`（保存策略及条件）。

- **修改和删除策略** 按钮  
  - 功能：打开策略管理弹窗，可对已保存策略进行重命名或删除。  
  - 弹窗内表格显示策略名称、条件摘要（ΔP、ΔWP、P-Tier）。  
  - 后端接口：  
    - 修改：`PUT /api/strategies/{name}`  
    - 删除：`DELETE /api/strategies/{name}`

---

### 2.3 其它条件卡片（FilterSection 其它条件部分）
> 与策略维度无关，叠加在策略之上的过滤条件

- **联赛**  
  - 多选列表（如 英超、西甲、德甲、意甲、法甲、中超）。  
  - 对应后端参数：`leagues`。

- **date_time**  
  - 下拉选项：显示数据库中实际的 `date_time` 字段值（如 `26024`、`26023`、`26022`）。  
  - 默认加载最新 3 期。  
  - 点击“获取实时数据”会刷新为最新 3 期。  
  - 对应后端参数：`dateTime`。

- **date_range**  
  - 日期范围选择器。  
  - 对应后端参数：`dateRange`。

---

### 2.4 排序方式卡片（SortSection 或 FilterSection 排序部分）
- **排序字段**  
  - 选项：`p_level`、`delta_wp`、`power_diff` 等。  
  - 对应后端参数：`sortBy`。

- **排序顺序**  
  - 选项：`asc`（升序）、`desc`（降序）。  
  - 对应后端参数：`sortOrder`。

- **说明文案**：此排序仅作用于筛选结果列表的比赛场次。

---

### 2.5 策略筛选卡片（StrategySection）
- **策略筛选下拉栏**  
  - 包含：已保存策略、示例策略、当前策略。  
  - 选择策略 → 加载对应条件并立即筛选（示例策略除外，仅加载条件不筛选）。

- **策略列表**  
  - 显示策略名称，可点击选择。  
  - 当前选中的策略在界面上有“已选”状态（逻辑控制，可加视觉标识）。

---

### 2.6 筛选统计卡片（StatsCard）
- **显示条件**：仅当用户选择了策略后才显示（无论是否有结果，场次数为 0 也显示）。  
- **内容**：统计符合条件场次等信息（具体字段依后端返回 `statistics` 而定）。

---

### 2.7 筛选结果卡片（ResultsSection）
- **显示条件**：仅当用户选择了策略后才显示。  
- **无结果时**：显示“没有符合场次”。  
- **有结果时**：分页列出比赛场次（主队、客队、比分、P级等）。  
- **分页参数**：`currentPage`、`pageSize`（前端控制）。  
- **导出按钮**：CSV、JSON、Excel（调用后端导出接口）。

---

## 3. 接口参数对照表

### 3.1 筛选接口参数
| 前端字段 | 后端参数 | 说明 |
|----------|----------|------|
| powerDiffs | powerDiffs | 实力等级差选项 |
| winPanDiffs | winPanDiffs | 赢盘等级差选项 |
| stabilityTiers | stabilityTiers | 一赔稳定性选项 |
| leagues | leagues | 联赛多选 |
| dateTime | dateTime | 日期时间值 |
| dateRange | dateRange | 日期范围 |
| sortBy | sortBy | 排序字段 |
| sortOrder | sortOrder | 排序顺序 |
| includeDerating | includeDerating | 是否包含降级处理（如有） |

### 3.2 其它接口
- 获取最新比赛数量：`GET /api/latest-matches-count` → `{ count, latestDate }`
- 获取可用 date_time 列表：`GET /api/available-date-times?limit=3` → `{ dates: string[] }`
- 保存策略：`POST /api/strategies` → `{ name, filters }`
- 修改策略：`PUT /api/strategies/{name}` → `{ name, filters }`
- 删除策略：`DELETE /api/strategies/{name}`
- 筛选结果：`POST /api/filter` → `{ totalResults, pagedResults, statistics }`
- 导出结果：`GET /api/export/{format}`（format: csv/json/excel）

---

## 4. 交互流程简述
1. **设置策略维度条件** → 点击“应用筛选” → 生成“当前策略” → 显示统计与结果。  
2. **设置其它条件**（联赛、date_time、date_range） → 与策略条件叠加过滤。  
3. **设置排序方式** → 作用于结果列表顺序。  
4. **选择已保存策略** → 立即加载条件并筛选。  
5. **示例策略** → 仅预览条件，不自动筛选。  
6. **获取实时数据** → 更新实时场次数和 date_time 下拉选项。
