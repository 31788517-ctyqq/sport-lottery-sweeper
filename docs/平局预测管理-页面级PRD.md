# 平局预测管理页面级PRD（闭环版）

## 1. 文档目标
- 将以下四个页面从“分散功能”整合为“可执行闭环”：
  - `/admin/draw-prediction/data-features`
  - `/admin/draw-prediction/training-evaluation`
  - `/admin/draw-prediction/model-deployment`
  - `/admin/draw-prediction/prediction-monitoring`
- 输出可直接进入开发排期的页面级需求，不涉及代码实现细节。

## 2. 约束与硬性要求
- 每个页面必须有可见数据，避免空页面：
  - `data-features`：保留现有已录入的17条特征，不删除、不覆盖。
  - `training-evaluation`：至少1条模拟训练任务。
  - `model-deployment`：至少1条模拟模型版本（可部署状态）。
  - `prediction-monitoring`：至少1条模拟预测结果。
- 闭环链路必须可追溯：`特征 -> 训练任务 -> 模型版本 -> 预测结果`。
- 任何页面操作必须有“去上游/去下游”入口，不能形成操作死路。

## 3. 业务闭环定义
1. 特征管理：维护并启用特征。
2. 训练评估：基于特征发起训练并得到评估结果。
3. 模型部署：从训练产物中选择候选模型并上线。
4. 预测监控：监控线上效果并触发再训练建议。

闭环完成标准：用户在四个页面内可完成一次完整流程，无需跳出模块或人工补链。

## 4. 页面级需求

## 4.1 Data Features（数据与特征管理）
### 页面定位
- 平局预测特征的唯一管理入口。

### 核心功能
- 特征列表查询、启停、编辑。
- 展示“被哪些训练任务引用”的引用关系（新增）。
- 提供“去训练评估”入口（新增）。

### 数据要求
- 保留现有17条特征。
- 不新增“占位脏数据”；若需演示，使用现有特征中的1条作为默认示例高亮。

### 交互补充
- 列表顶部显示：
  - 已启用特征数
  - 被训练任务引用的特征数
  - 最近更新时间

## 4.2 Training Evaluation（模型训练与评估）
### 页面定位
- 训练任务发起与评估中心。

### 核心功能
- 新建训练任务（任务名、算法、超参、特征集）。
- 查看训练状态、日志、评估指标。
- 达标任务支持“推送至模型部署”（新增按钮）。

### 数据要求
- 至少1条模拟训练任务，状态建议为 `success`，含指标：
  - `accuracy/precision/recall/f1_score/auc`

### 交互补充
- 增加“去模型部署”跳转（携带 `training_job_id`）。
- 增加“数据来源说明”：本任务使用的特征ID列表。

## 4.3 Model Deployment（模型管理与部署）
### 页面定位
- 候选模型选择、上线、回滚中心。

### 核心功能
- 模型版本列表、部署、回滚。
- 模型详情中展示来源训练任务与关键指标。
- 增加“去预测监控”入口（新增）。

### 数据要求
- 至少1条模拟模型版本：
  - 关联模拟训练任务
  - 含版本号、指标、状态（建议 `active` 或可部署状态）

### 交互补充
- 若无可部署模型，提示“先去训练评估创建任务”并给跳转按钮。
- 部署成功后落地部署时间并记录操作人（规划项）。

## 4.4 Prediction Monitoring（预测服务与监控）
### 页面定位
- 线上预测表现监控与再训练入口。

### 核心功能
- 统计总预测、命中数、命中率、待确认。
- 按时间过滤查看预测记录。
- 增加“触发再训练建议”入口（新增）。

### 数据要求
- 至少1条模拟预测记录：
  - `match_id`
  - `predicted_draw_prob`
  - `actual_result`（建议为 `draw`，便于命中率验证）
  - `predicted_at/match_time`

### 交互补充
- 记录应可回溯到模型版本ID（规划项，可放在 `prediction_meta`）。
- 提供“返回模型部署”入口。

## 5. 页面联动设计
- `data-features -> training-evaluation`：带 `feature_ids` 默认值。
- `training-evaluation -> model-deployment`：带 `training_job_id`。
- `model-deployment -> prediction-monitoring`：带 `model_version_id`。
- `prediction-monitoring -> training-evaluation`：带“再训练建议参数草稿”。

## 6. 非功能要求
- 首屏可用性：四页首屏均不得为空。
- 可追溯性：任意一条预测可反查模型版本与训练任务。
- 可运维性：模拟数据可重复初始化，不影响真实业务数据。

## 7. 交付拆分（建议）
1. 第一批：四页模拟数据接入 + 页面空态治理。
2. 第二批：页面间联动入口与参数透传。
3. 第三批：闭环追溯信息与再训练建议链路。

## 8. 验收门槛
- 四个页面均有数据可见。
- `data-features` 仍为17条。
- 完成一次“特征 -> 训练 -> 部署 -> 监控”的可操作演示路径。
