## 2. 100qiu数据获取流程设计

### 2.1 API接口详情

- **API地址**: `https://m.100qiu.com/api/dcListBasic`
- **参数**: `dateTime` - 指定日期时间参数，默认为最新数据
- **返回格式**: JSON 格式的比赛数据

### 2.2 数据获取流程

1. **配置数据源**:
   - 创建100qiu数据源配置，包括URL、参数设置等
   - 设置更新频率和错误处理策略

2. **发起请求**:
   - 构造带参数的API请求
   - 处理HTTP响应和错误情况

3. **数据解析**:
   - 解析返回的JSON数据
   - 映射字段到内部数据模型

4. **数据存储**:
   - 按照足球设计模型存储数据
   - 避免重复数据插入

5. **结果反馈**:
   - 返回获取数据的数量和样本数据
   - 记录错误日志和成功率

### 2.3 数据完整性保证

- **字段完整性**: 确保所有必要字段都被提取和存储
- **数据条数完整性**: 通过比较API返回总数和实际存储数来验证
- **去重机制**: 通过唯一标识符避免重复数据存储

## 3. 数据源管理模块改进

### 3.1 后端改进

#### 3.1.1 新增API端点

创建了专门的100qiu数据源API (`/backend/api/v1/data_source_100qiu.py`)，包含以下功能：

- `POST /`: 创建100qiu数据源
- `GET /{source_id}`: 获取指定ID的100qiu数据源
- `PUT /{source_id}`: 更新100qiu数据源配置
- `DELETE /{source_id}`: 删除100qiu数据源
- `GET /`: 获取100qiu数据源列表
- `POST /{source_id}/test`: 测试100qiu数据源连接
- `POST /{source_id}/fetch`: 从100qiu API获取数据

#### 3.1.2 数据模型

使用现有的 [DataSource](file://c:\Users\11581\Downloads\sport-lottery-sweeper\backend\models\data_sources.py#L14-L84) 模型，添加了对100qiu特定配置的支持：

- `config` 字段存储 `date_time` 参数和更新频率
- `field_mapping` 字段用于自定义字段映射
- `last_error` 和 `last_error_time` 字段用于错误追踪

#### 3.1.3 数据解析逻辑

在 `parse_match_from_100qiu` 函数中实现了灵活的数据解析：

- 处理多种可能的字段名称变体
- 时间格式转换和标准化
- 自动为100qiu数据生成唯一ID前缀
