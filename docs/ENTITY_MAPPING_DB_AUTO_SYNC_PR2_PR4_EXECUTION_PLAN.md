# 实体映射 DB 自动补全 PR-2 ~ PR-4 执行计划

## 1. 范围

- PR-2：映射质量治理（去噪、冲突识别、审核状态）
- PR-3：官方信息自动补全闭环（自动补全 + 失败重试 + 状态可见）
- PR-4：页面联调与运营可视化（系统页/数据源页统一）

不包含：
- 新模型训练、深度语义匹配模型上线。

## 2. 里程碑

| 里程碑 | 目标 | 产物 | 门禁 |
|---|---|---|---|
| M1 (PR-2 完成) | 映射质量可治理 | 冲突列表、审核接口、质量字段 | API 与页面回归通过 |
| M2 (PR-3 完成) | 官方信息自动补全闭环 | enrich 任务链路、状态接口 | 自动补全成功率达标 |
| M3 (PR-4 完成) | 运维可视化闭环 | 三页联调、统一文案/状态 | 无关键 4xx/5xx |

## 3. 风险清单与应对

| 风险 | 等级 | 应对 |
|---|---|---|
| 别名脏数据导致误合并 | 高 | 引入冲突待审，不自动覆盖人工结果 |
| 自动补全外部源不稳定 | 中 | 限频+重试+失败状态可视化 |
| 前端改造影响旧入口 | 中 | 保持旧接口兼容，新增入口灰度开关 |

## 4. 验收口径

- 功能：主链路可跑通（同步/补全/审核）。
- 稳定性：核心接口 `4xx/5xx < 1%`。
- 运维：失败可见、可重试、可回滚。

## 5. 回滚策略

- 统一回滚到 PR-1 基线：
  - 关闭 PR-2/PR-3 新开关；
  - 保留 `mappings/*` 与 `sync/*` 现有行为；
  - 页面隐藏新增入口。
