# 爬虫源配置管理功能开发测试说明

## 1. 开发说明
- **前端框架**：Vue 3 + Element Plus  
- **路由**：`/admin/crawler/source-config` 嵌入主布局右侧窗口  
- **状态管理**：目前用本地 ref，可扩展 Pinia 存储树与配置状态  
- **功能交互**：
  - 所有按钮均绑定对应方法，含校验与提示
  - 新增/复制/删除会修改左侧树结构
  - 保存/导入/导出/回滚需接入后端 API（代码中标记 `TODO`）
- **文件处理**：导入支持 `.json/.yaml/.yml/.py`，导出为 `.json` 文件

## 2. 测试说明

### 2.1 功能测试
| 功能 | 测试步骤 | 预期结果 |
|------|----------|----------|
| 新增源 | 点击“新增源”，输入名称与ID，确认 | 树中新增节点，可选中编辑 |
| 复制配置 | 选中节点，点“复制配置” | 树中新增同名副本节点 |
| 删除 | 选中非全局节点，点“删除”，确认 | 节点从树中移除，清空右侧配置 |
| 保存 | 修改右侧 JSON，点“保存” | 提示保存成功（需后端支持） |
| 测试连接 | 选中节点，点“测试连接” | 提示连接成功/失败 |
| 导入 | 点“导入”，选择合法 JSON 文件 | 右侧配置替换为文件内容 |
| 导出 | 选中节点，点“导出” | 下载对应配置文件 |
| 回滚 | 点“回滚”，选版本确认 | 配置恢复，提示回滚成功 |

### 2.2 异常测试
- JSON 格式错误时保存 → 提示错误  
- 删除全局参数 → 提示不可删除  
- 未选节点进行操作 → 提示先选择数据源  
- 导入非支持格式或损坏文件 → 提示解析失败  

### 2.3 接口联调（后端配合）
- `GET /api/crawler/source-config/:id` 获取配置  
- `POST /api/crawler/source-config` 保存新配置（含版本记录）  
- `GET /api/crawler/source-config/:id/versions` 获取历史版本  
- `POST /api/crawler/source-config/:id/rollback` 回滚  
- `POST /api/crawler/source-config/import` 导入  
- `GET /api/crawler/source-config/export?format=json` 导出
