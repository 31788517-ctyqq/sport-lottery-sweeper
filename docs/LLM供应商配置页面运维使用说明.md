# LLM供应商配置页面运维使用说明

## 概述

LLM供应商配置页面是系统管理远程AI服务提供商的核心功能模块。运维人员可以通过此页面配置、管理和监控各种LLM供应商（如OpenAI、Anthropic、Google等），确保系统能够灵活调用不同的AI服务。

## 功能特性

### 1. 供应商管理
- **供应商列表**：查看所有已配置的LLM供应商
- **供应商详情**：查看供应商的配置信息、健康状态、使用统计
- **供应商筛选**：按供应商类型、健康状态、启用状态进行筛选
- **供应商搜索**：按名称快速搜索供应商

### 2. 供应商配置
- **添加供应商**：配置新的LLM供应商
- **编辑供应商**：修改现有供应商配置
- **删除供应商**：移除不再使用的供应商
- **启用/禁用**：临时启用或禁用供应商

### 3. 健康监控
- **健康状态**：实时显示供应商的连接状态
- **连接测试**：手动测试供应商的连接性
- **使用统计**：查看请求数、成功率、响应时间等指标
- **成本监控**：跟踪API使用成本

### 4. 高级配置
- **优先级设置**：设置供应商的调用优先级（1-10，值越小优先级越高）
- **限流配置**：配置每分钟最大请求数
- **超时设置**：设置API调用超时时间
- **重试策略**：配置失败重试机制

## 操作指南

### 访问页面
1. 登录系统管理后台
2. 导航至 `AI服务管理` → `远程AI服务` 页面
3. 或直接访问：`http://localhost:3000/admin/ai-services/remote`

### 添加新供应商
1. 点击页面右上角的 `添加远程服务` 按钮
2. 填写供应商信息：
   - **供应商名称**：自定义名称（唯一标识）
   - **供应商类型**：选择供应商类型（OpenAI、Anthropic等）
   - **API密钥**：输入供应商的API密钥
   - **基础URL**：（可选）自定义API端点
   - **默认模型**：指定默认调用的模型
   - **优先级**：设置调用优先级（1-10）
   - **每分钟请求限制**：设置限流阈值
   - **超时时间**：设置API调用超时（秒）
3. 点击 `确定` 保存配置

### 编辑供应商
1. 在供应商列表中找到要编辑的供应商
2. 点击该行右侧的 `编辑` 按钮
3. 修改需要更新的配置信息
   - **注意**：API密钥字段留空表示不更新密钥
4. 点击 `确定` 保存修改

### 测试供应商连接
1. 在供应商列表中找到要测试的供应商
2. 点击该行右侧的 `测试` 按钮
3. 系统将发送测试请求验证连接性
4. 测试结果将显示在页面右上角

### 删除供应商
1. 在供应商列表中找到要删除的供应商
2. 点击该行右侧的 `删除` 按钮
3. 确认删除操作
4. **注意**：删除操作不可恢复，请谨慎操作

### 筛选和搜索
1. **按名称搜索**：在搜索框中输入供应商名称关键词
2. **按类型筛选**：使用供应商类型下拉框筛选
3. **按状态筛选**：使用健康状态下拉框筛选
4. **重置筛选**：点击 `重置` 按钮清除所有筛选条件

### 查看统计信息
1. 供应商列表显示以下统计信息：
   - **总请求数**：该供应商的总调用次数
   - **成功请求数**：成功调用的次数
   - **失败请求数**：调用失败的次数
   - **本月成本**：当前月的API使用成本
   - **健康状态**：实时连接状态
   - **最后检查时间**：最近一次健康检查的时间

## 故障排除

### 常见问题

#### 1. 供应商连接失败
**可能原因**：
- API密钥无效或过期
- 网络连接问题
- 供应商服务不可用
- 基础URL配置错误

**解决方案**：
1. 检查API密钥是否正确
2. 测试网络连接是否正常
3. 检查供应商服务状态
4. 验证基础URL配置

#### 2. 请求超时
**可能原因**：
- 网络延迟过高
- 供应商响应慢
- 超时时间设置过短

**解决方案**：
1. 适当增加超时时间设置
2. 检查网络状况
3. 联系供应商技术支持

#### 3. 成本异常
**可能原因**：
- API调用频率过高
- 模型使用成本变化
- 配置错误导致重复调用

**解决方案**：
1. 检查限流配置是否合理
2. 查看详细调用日志
3. 联系财务部门核实账单

#### 4. 健康状态显示异常
**可能原因**：
- 健康检查失败
- 数据库连接问题
- 监控服务异常

**解决方案**：
1. 手动测试供应商连接
2. 检查数据库连接状态
3. 重启监控服务

### 监控指标

运维人员应定期监控以下指标：

#### 1. 健康状态
- **目标**：所有供应商的健康状态应为"健康"
- **监控频率**：每小时检查一次
- **告警阈值**：连续3次检查失败

#### 2. 成功率
- **目标**：成功率 ≥ 99%
- **监控频率**：每天统计一次
- **告警阈值**：成功率 < 95%

#### 3. 响应时间
- **目标**：平均响应时间 ≤ 5秒
- **监控频率**：每小时统计一次
- **告警阈值**：平均响应时间 > 10秒

#### 4. 成本控制
- **目标**：月度成本在预算范围内
- **监控频率**：每天统计一次
- **告警阈值**：日成本超过月度预算的5%

## 紧急操作流程

### 1. 供应商故障
**场景**：主要供应商服务不可用

**操作流程**：
1. 立即切换到备用供应商
2. 联系供应商技术支持
3. 通知相关业务部门
4. 记录故障时间和影响范围

### 2. 成本超支
**场景**：API使用成本超出预算

**操作流程**：
1. 立即启用限流策略
2. 暂停非关键业务调用
3. 分析成本异常原因
4. 调整预算和告警阈值

### 3. 安全事件
**场景**：API密钥泄露

**操作流程**：
1. 立即禁用相关供应商
2. 生成新的API密钥
3. 更新所有相关配置
4. 进行安全审计
