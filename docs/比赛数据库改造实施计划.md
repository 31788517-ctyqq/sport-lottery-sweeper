# 比赛数据库改造实施计划

## 项目概述

### 项目背景
随着体育彩票分析系统的发展，现有比赛数据库面临以下挑战：
1. **多数据源整合困难**：不同数据源（如100qiu、500等）的字段命名、格式不一致
2. **数据质量参差不齐**：缺乏统一的数据质量检查和标准化机制
3. **扩展性受限**：现有表结构难以支持新数据源的快速接入
4. **分析能力不足**：缺乏跨数据源的统一分析视图

### 改造目标
1. **建立统一数据模型**：支持多数据源的比赛数据整合
2. **提升数据质量**：建立数据验证、清洗和标准化流程
3. **增强扩展性**：设计灵活架构，支持未来数据源快速接入
4. **支持高级分析**：提供跨数据源的统一分析视图

### 项目范围
- **核心改造**：比赛相关表结构重构
- **数据迁移**：现有比赛数据的标准化迁移
- **接口适配**：前后端接口的兼容性调整
- **质量监控**：数据质量检查机制建立

## 现状分析

### 现有数据库结构
```sql
-- 当前主要比赛表结构（backend/models/match.py）
CREATE TABLE matches (
    id INTEGER PRIMARY KEY,
    match_identifier VARCHAR(50) UNIQUE,
    home_team_id INTEGER,
    away_team_id INTEGER,
    league_id INTEGER,
    match_date DATE,
    match_time TIME,
    scheduled_kickoff DATETIME,
    status VARCHAR(20),
    home_score INTEGER,
    away_score INTEGER,
    -- 缺少数据源标识和版本管理
);
```

### 主要问题识别
1. **数据源缺失**：无法区分不同数据源的同一场比赛
2. **字段不统一**：不同数据源字段命名和格式差异大
3. **版本管理空白**：数据更新无法追踪历史版本
4. **质量监控缺失**：缺乏数据质量检查和错误记录

### 技术债务
- 现有API依赖当前表结构
- 前端展示逻辑基于现有字段
- 爬虫系统输出格式固定

## 改造方案设计

### 核心架构原则
1. **向后兼容**：保持现有API接口不变
2. **渐进式改造**：分阶段实施，降低风险
3. **数据可追溯**：保留数据来源和版本信息
4. **灵活扩展**：支持新数据源快速接入

   ```python
   class DataQualityChecker:
       def __init__(self):
           self.checks = [
               self.check_required_fields,
               self.check_score_consistency,
               self.check_team_mapping,
               self.check_time_validity,
               self.check_odds_range
           ]
       
       def check_match_data(self, match_data: Dict) -> List[QualityIssue]:
           issues = []
           for check in self.checks:
               result = check(match_data)
               if result:
                   issues.extend(result)
           return issues
       
       def check_required_fields(self, data):
           required = ['match_time', 'home_team', 'away_team', 'league']
           missing = [field for field in required if not data.get(field)]
           if missing:
               return [QualityIssue(
                   type='required_field_missing',
                   level='error',
                   message=f'缺少必填字段: {missing}'
               )]
           return []
       
       def check_score_consistency(self, data):
           # 检查比分逻辑一致性
           if data.get('home_score') and data.get('away_score'):
               if data['home_score'] < 0 or data['away_score'] < 0:
                   return [QualityIssue(
                       type='invalid_score',
                       level='error',
                       message='比分不能为负数'
                   )]
           return []
   ```

4. **创建数据清洗管道**
   ```python
   class DataCleaningPipeline:
       def __init__(self):
           self.steps = [
               FieldExtractor(),
               ValueNormalizer(),
               DuplicateChecker(),
               QualityValidator(),
               VersionTracker()
           ]
       
       def process(self, raw_data: Dict, source: str) -> ProcessedData:
           context = {'source': source, 'original_data': raw_data}
           result = raw_data
           
           for step in self.steps:
               result = step.execute(result, context)
               if not result:
                   raise ProcessingError(f"步骤 {step.__class__.__name__} 失败")
           
           return ProcessedData(
               standard_fields=result['standard'],
               source_attributes=result['source_specific'],
               quality_score=result.get('quality_score', 100),
               issues=result.get('issues', [])
           )
   ```

   ```sql
   -- 比赛分析视图
   CREATE VIEW match_analysis_view AS
   SELECT 
       m.id,
       m.match_identifier,
       m.data_source,
       m.source_match_id,
       m.match_date,
       m.match_time,
       m.scheduled_kickoff,
       m.status,
       m.home_score,
       m.away_score,
       m.data_version,
       
       -- 标准化字段（从JSON中提取）
       JSON_EXTRACT(m.source_attributes, '$.standardized.home_team') AS home_team,
       JSON_EXTRACT(m.source_attributes, '$.standardized.away_team') AS away_team,
       JSON_EXTRACT(m.source_attributes, '$.standardized.league') AS league,
       
       -- 数据质量指标
       (SELECT COUNT(*) FROM data_quality_logs dql 
        WHERE dql.match_id = m.id AND dql.resolved = FALSE) AS unresolved_issues,
       
       -- 数据源可靠性评分
       (SELECT reliability_score FROM data_source_reliability dsr
        WHERE dsr.source_code = m.data_source) AS source_reliability,
       
       -- 最后更新时间
       m.updated_at
   FROM matches m
   WHERE m.status IN ('scheduled', 'in_progress', 'finished')
   ORDER BY m.scheduled_kickoff DESC;
   
   -- 数据源对比视图
   CREATE VIEW data_source_comparison_view AS
   SELECT 
       m.match_identifier,
       GROUP_CONCAT(DISTINCT m.data_source) AS available_sources,
       COUNT(DISTINCT m.data_source) AS source_count,
       
       -- 各数据源关键字段对比
       MAX(CASE WHEN m.data_source = 'source_100qiu' 
           THEN JSON_EXTRACT(m.source_attributes, '$.odds.home_win') END) AS odds_100qiu,
       MAX(CASE WHEN m.data_source = 'source_500' 
           THEN JSON_EXTRACT(m.source_attributes, '$.odds.home') END) AS odds_500,
       
       -- 一致性检查
       CASE WHEN COUNT(DISTINCT JSON_EXTRACT(m.source_attributes, '$.standardized.home_score')) = 1
           THEN 'consistent' ELSE 'inconsistent' END AS score_consistency
   FROM matches m
   GROUP BY m.match_identifier
   HAVING COUNT(DISTINCT m.data_source) > 1;
   ```

2. **实现数据融合算法**
   ```python
   class DataFusionEngine:
       def __init__(self, config):
           self.config = config
           self.conflict_resolvers = {
               'score': self.resolve_score_conflict,
               'odds': self.resolve_odds_conflict,
               'time': self.resolve_time_conflict
           }
       
       def fuse_match_data(self, match_identifier: str) -> FusedMatchData:
           # 获取所有数据源的该场比赛数据
           sources_data = self.fetch_all_sources(match_identifier)
           
           if not sources_data:
               return None
           
           # 基础字段融合（取最可靠数据源）
           fused = self.fuse_basic_fields(sources_data)
           
           # 处理冲突字段
           for field in ['home_score', 'away_score', 'match_time', 'odds']:
               if self.has_conflict(sources_data, field):
                   fused[field] = self.resolve_conflict(sources_data, field)
           
           # 计算置信度
           fused['confidence_score'] = self.calculate_confidence(sources_data)
           fused['source_contributions'] = self.get_source_contributions(sources_data)
           
           return fused
       
       def resolve_score_conflict(self, sources_data):
           # 策略：取最新数据源，或多数一致的值
           scores = {}
           for source, data in sources_data.items():
               score_key = f"{data.get('home_score')}-{data.get('away_score')}"
               scores[score_key] = scores.get(score_key, 0) + 1
           
           # 取出现次数最多的比分
           most_common = max(scores.items(), key=lambda x: x[1])
           home_score, away_score = most_common[0].split('-')
           
           return {
               'home_score': int(home_score),
               'away_score': int(away_score),
               'confidence': most_common[1] / len(sources_data)
           }
   ```

4. **开发分析API接口**
   ```python
   # 分析API端点
   @router.get("/api/v1/analysis/matches/comparison")
   async def get_match_comparison(
       match_identifier: str = Query(..., description="比赛标识"),
       include_details: bool = Query(False, description="包含详情")
   ):
       """
       获取多数据源比赛数据对比
       """
       # 获取所有数据源的该场比赛
       sources_data = await match_service.get_by_identifier_from_all_sources(match_identifier)
       
       if not sources_data:
           raise HTTPException(status_code=404, detail="比赛未找到")
       
       # 基础对比信息
       comparison = {
           'match_identifier': match_identifier,
           'available_sources': list(sources_data.keys()),
           'source_count': len(sources_data),
           'consistency_analysis': {}
       }
       
       # 字段一致性分析
       for field in ['home_score', 'away_score', 'match_time', 'home_team', 'away_team']:
           values = []
           for source, data in sources_data.items():
               if field in data:
                   values.append({
                       'source': source,
                       'value': data[field],
                       'timestamp': data.get('updated_at')
                   })
           
           if values:
               comparison['consistency_analysis'][field] = {
                   'values': values,
                   'is_consistent': len(set(v['value'] for v in values)) == 1,
                   'recommended_value': self.get_recommended_value(values)
               }
       
       # 数据融合结果（如果启用）
       if include_details:
           fusion_engine = DataFusionEngine()
           fused_data = fusion_engine.fuse_match_data(match_identifier)
           comparison['fused_data'] = fused_data
       
       return comparison
   ```
