# 平局预测管理接口清单（闭环与模拟数据版）

## 1. 范围说明
- 覆盖页面：
  - `data-features`
  - `training-evaluation`
  - `model-deployment`
  - `prediction-monitoring`
- 本文档用于研发联调，不包含代码实现。

## 2. 数据对象与主链路
- `DrawFeature`（特征）
- `DrawTrainingJob`（训练任务）
- `DrawModelVersion`（模型版本）
- `DrawPredictionResult`（预测结果）

主链路：`feature_ids -> training_job_id -> model_version_id -> prediction_result`

## 3. 现有接口（可直接使用）

## 3.1 Data Features
1. `GET /api/draw-prediction/features`
2. `POST /api/draw-prediction/features`
3. `PUT /api/draw-prediction/features/{feature_id}`
4. `DELETE /api/draw-prediction/features/{feature_id}`

## 3.2 Training Evaluation
1. `GET /api/draw-prediction/training-jobs`
2. `POST /api/draw-prediction/training-jobs`
3. `GET /api/draw-prediction/training-jobs/{job_id}/logs`
4. `PUT /api/draw-prediction/training-jobs/{job_id}/status`

## 3.3 Model Deployment
1. `GET /api/draw-prediction/models`
2. `POST /api/draw-prediction/models/{model_id}/deploy`
3. `POST /api/draw-prediction/models/{model_id}/rollback`

## 3.4 Prediction Monitoring
1. `GET /api/draw-prediction/predictions`

## 4. 必补接口（闭环增强）

## 4.1 页面联动与摘要接口
1. `GET /api/draw-prediction/training-jobs/summary`
- 用途：`model-deployment` 顶部训练状态概览。
- 返回：`pending/running/success/failed` 聚合计数。

2. `GET /api/draw-prediction/models/{model_id}/trace`
- 用途：追溯模型来源。
- 返回：`training_job_id/feature_set_ids/version_tag/deployed_at`。

3. `GET /api/draw-prediction/predictions/summary`
- 用途：`prediction-monitoring` 统计卡片独立读取。
- 返回：`total/hits/accuracy/pending`。

## 4.2 再训练闭环接口
1. `POST /api/draw-prediction/retrain-drafts`
- 用途：从监控页生成再训练草稿。
- 入参：`model_version_id`、时间窗口、触发原因。
- 出参：`draft_id`、默认训练参数。

2. `POST /api/draw-prediction/retrain-drafts/{draft_id}/submit`
- 用途：将草稿转换为训练任务。
- 出参：`training_job_id`。

## 4.3 模拟数据初始化接口（非生产建议）
1. `POST /api/draw-prediction/mock/bootstrap`
- 用途：一键创建四页可见模拟数据。
- 规则：
  - 保留 `data-features` 现有17条，不清理、不覆盖。
  - 若训练/模型/预测为空，则各创建1条模拟记录。
  - 幂等：重复调用不重复插入。
- 出参：创建结果摘要。

2. `GET /api/draw-prediction/mock/bootstrap/status`
- 用途：查看模拟数据初始化状态。

## 5. 页面到接口映射

| 页面 | 首屏加载 | 主要动作 | 联动跳转参数 |
|---|---|---|---|
| data-features | `GET /features` | 增删改启停 | `feature_ids` |
| training-evaluation | `GET /training-jobs` | 创建任务、看日志、更新状态 | `training_job_id` |
| model-deployment | `GET /models` + `GET /training-jobs/summary` | 部署、回滚 | `model_version_id` |
| prediction-monitoring | `GET /predictions` + `GET /predictions/summary` | 时间筛选、生成再训练草稿 | `draft_id` |

## 6. 关键字段规范（统一口径）
- 训练状态：`pending/running/success/failed`
- 模型状态：`inactive/active`（兼容 `deployed` 显示）
- 预测结果：`draw/home/away/null`
- 指标字段统一：`accuracy/precision/recall/f1_score/auc`

## 7. 模拟数据约束（执行标准）
1. `data-features`：必须保留17条现有数据。
2. `training-evaluation`：至少1条任务，且带完整指标。
3. `model-deployment`：至少1条版本，关联该任务。
4. `prediction-monitoring`：至少1条预测，能参与统计卡片计算。

## 8. 错误码与降级建议
- `404`：上游对象不存在时，页面显示“去上游创建”按钮。
- `422`：参数校验失败，返回字段级错误。
- `500`：系统错误，前端保留最近成功数据并提示重试。
