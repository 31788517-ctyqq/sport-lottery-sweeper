# 情报采集管理方案决策对比表（V1）
更新时间：2026-02-19

## 1. 核心对比（投入、周期、风险、收益）
| 维度 | 降本可落地版（MVP） | 全功能版（目标态） |
|---|---|---|
| 目标 | 4-6 周打通闭环，先上线可用 | 形成完整情报中台，支撑长期运营 |
| 功能范围 | 3 个数据源、5 类情报、钉钉文本推送 | 10+ 数据源、全情报类型、个性化+PDF+复盘 |
| 技术复杂度 | 中 | 高 |
| 人力投入（建议） | 3-5 人（后端2、前端1、测试/运维1） | 7-10 人（后端4+、前端2、测试1-2、运维1） |
| 基础设施投入 | 低-中（单服务+worker+redis） | 中-高（多微服务+K8s+监控治理） |
| 上线周期 | 4-6 周 | 3-6 个月（分阶段） |
| 交付确定性 | 高 | 中 |
| 主要风险 | 源站解析不稳定、规则精度初期一般 | 项目周期长、跨团队协同复杂、维护成本高 |
| 风险可控性 | 高（范围可控，快速迭代） | 中（依赖治理体系成熟度） |
| 短期收益 | 快速验证价值，尽快产生用户反馈 | 短期收益较慢 |
| 长期收益 | 中（可持续扩展） | 高（规模化、精细化运营能力） |
| 推荐场景 | 需要尽快上线、预算有限、先验证产品价值 | 预算充足、团队成熟、追求中长期壁垒 |

## 3. 推荐决策
1. 当前优先选择：**降本可落地版（MVP）先上线**。  
2. 决策原则：先闭环、后增强；先稳定、后复杂。  
3. 升级条件（满足后进入目标态）：
- 连续 2-4 周任务成功率 >= 90%
- 推送送达率 >= 98%
- 用户反馈“有用率”持续提升
- 团队具备独立运维多服务能力

## 4. 建议实施路径（两步走）
1. 第一步（0-2个月）：MVP 上线 + 指标观测 + 小步迭代。
2. 第二步（2-6个月）：按优先级逐步并入全功能能力（变化触发、PDF、复盘、更多数据源）。
