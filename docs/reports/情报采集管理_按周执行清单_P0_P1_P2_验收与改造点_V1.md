# 情报采集管理按周执行清单（P0/P1/P2）+ 验收标准 + 接口/表改造点 V1

> 适用页面：`/admin/intelligence/collection`  
> 目标：提升“抓取成功率 + 命中质量 + 页面可观测性 + 上线可运维性”  
> 版本：V1  
> 日期：2026-02-20

## 1. 核心目标与量化指标

| 指标 | 定义 | 当前问题 | 阶段目标 |
|---|---|---|---|
| 抓取成功率 | HTTP成功请求数 / 总请求数 | 连接超时、反爬导致失败 | P0后 >= 95% |
| 解析成功率 | 详情页解析成功数 / HTTP成功请求数 | 命中目录页、摘要页较多 | P0后 >= 80% |
| 比赛匹配率 | 通过比赛硬门槛条目数 / 解析成功数 | 球队别名、时间窗误判 | P1后 >= 60% |
| 可用情报率 | quality_status=accepted / 匹配条目数 | 噪声高、链接不准 | P1后 >= 50% |
| 运营抽检准确率 | 人工抽检“相关且可读”比例 | 标题/摘要可读性差 | P2后 >= 85% |

## 2. 周期规划（按周映射 P0/P1/P2）

- W1-W2：P0（先稳）
- W3-W4：P1（再准）
- W5-W6：P2（好用与可运营）

## 3. W1-W6 可勾选执行清单

## W1（P0）：采集链路稳定化（任务、超时、重试）

| 勾选 | 工作项 | 验收标准 | 接口改造点 | 表字段改造点 |
|---|---|---|---|---|
| [ ] | 任务创建改为“快速返回 + 后台执行” | `POST /tasks` 在 2s 内返回，前端不再因 15s 超时报错 | 增强 `POST /api/v1/admin/intelligence/collection/tasks` 返回 `accepted/running` 与 `task_id`；增强 `GET /tasks/{task_id}` 返回阶段进度 | `intel_collection_tasks` 新增 `request_payload_json`、`config_snapshot_json`、`queue_job_id`、`progress_percent` |
| [ ] | 子任务按来源/比赛分片执行 | 单任务失败不拖垮全局，失败源可重试 | 增强 `GET /tasks/{task_id}/subtasks` 返回分片统计 | `intel_collection_match_subtasks` 新增 `candidate_count`、`parsed_count`、`matched_count`、`accepted_count`、`blocked_count` |
| [ ] | 错误分级重试（timeout/429/5xx） | 重试后整体抓取成功率 >= 95% | `POST /tasks/{task_id}/retry` 支持按 source/match 重试参数（可选） | `intel_collection_match_subtasks` 新增 `retry_policy`、`last_http_status`、`last_error_code` |

| [ ] | 任务日志结构化 | 可追踪到“哪个来源/哪条候选/为何失败” | 增强 `GET /tasks/{task_id}/logs` 支持 `source`、`level`、`stage` 过滤 | 新增 `intel_task_run_logs`（可选）或在 `logs_json` 中标准化字段 |

## W2（P0）：链接精准定位（列表页 -> 详情页）

| 勾选 | 工作项 | 验收标准 | 接口改造点 | 表字段改造点 |
|---|---|---|---|---|
| [ ] | 为重点来源加专用解析器（500w/ttyingqiu/weibo/tencent） | 详情页命中率明显高于通用解析器（>=80%） | 新增 `GET /sources` 返回 `parser_status`；新增 `POST /debug/match-candidates` 通用调试字段 | `intel_collection_items` 新增 `source_parser`、`detail_type`、`debug_trace_json` |
| [ ] | 强制二跳：列表页仅采候选，不直接入库 | 入库条目 `detail_type=article_detail` 占比 >= 70% | `POST /debug/match-candidates` 返回 `candidate_url`、`detail_url`、`score` | `intel_collection_items` 新增 `canonical_url`、`detail_url`、`dedupe_hash` |
| [ ] | 目录/低质量固定页治理（黑名单+降权） | 黑名单命中页不入库；降权页保留候选但不优先 | 新增 `GET/PUT /settings/source-rules`（新增） | 新增 `intel_source_parser_rules`（黑名单、白名单、降权规则） |

| [ ] | 发布时间提取标准化 | 时间可用率 >= 85%，时区一致 | 在 item 返回里增加 `published_at_parse_status`（可选） | `intel_collection_items` 新增 `published_at_raw`、`published_tz` |
