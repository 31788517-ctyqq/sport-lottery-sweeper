# 后台管理系统按钮跳转页面功能 - 全链路测试清单

## 1️⃣ 环境准备
- [ ] 确保后端服务已启动（默认 `http://localhost:8001`），并能返回正常接口数据。  
- [ ] 前端项目依赖已安装：`npm install` 或 `yarn install`。  
- [ ] 启动前端开发服务器：`npm run dev` 或 `yarn dev`。  
- [ ] 清除浏览器缓存或使用无痕模式，避免旧代码干扰。  

## 2️⃣ 路由解析测试
- [ ] 打开浏览器控制台，观察启动时 **无 Vite 预转换错误**（如 `invalid JS syntax`）。  
- [ ] 在 `router/index.js` 中确认已引入 `admin-routes.js`，且 `/admin` 路由只定义一次（无重复）。  
- [ ] 在 Vue DevTools 的 Routes 面板中，能看到完整的 `/admin` 及其子路由结构。  

## 3️⃣ 登录与权限验证测试
- [ ] 访问 `http://localhost:xxxx/admin/dashboard`，未登录时应自动跳转到 `/admin/login`（路由守卫生效）。  
- [ ] 使用有效账号登录，确认能进入后台首页。  
- [ ] 手动清除 `token`（如 localStorage/sessionStorage），再次访问受保护页面，应跳回登录页。  
- [ ] 在 `request.js` 的 401 拦截器中测试：后端返回 401 时，前端应提示并跳转登录页。

## 5️⃣ 接口请求与页面渲染测试
- [ ] 在任意有权限的页面，打开浏览器 Network 面板，刷新页面或触发数据请求。  
- [ ] 确认请求头带 `Authorization: Bearer <token>`。  
- [ ] 模拟后端返回 `code !== 0`，检查页面是否提示业务错误且不崩溃。  
- [ ] 模拟后端返回 401，检查是否跳转登录页。  
- [ ] 断网或后端不可达时，检查是否提示“网络连接失败”。  

## 6️⃣ 角色权限测试
- [ ] 使用 `admin` 角色登录，确认所有菜单可见。  
- [ ] 使用 `manager` 角色登录（如果有），检查是否只能看到 `roles: ['admin','manager']` 的菜单。  
- [ ] 使用普通角色或无权限角色登录，检查受限页面是否 403 或不可见。  

## 7️⃣ 回归检查
- [ ] 检查浏览器控制台无红色错误日志。  
- [ ] 检查网络请求均为 200 或预期的业务 `code: 0`。  
- [ ] 检查页面样式、交互正常，按钮跳转流畅。  

> ✅ 完成以上清单后，即可确认后台管理系统的按钮跳转页面功能已全面修复并稳定运行。
