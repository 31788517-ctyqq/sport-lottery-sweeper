# 北单三维筛选器移动端交互原型

## 一、设计目标
- 将现有桌面端北单三维筛选器功能完整适配到移动端
- 优化小屏幕下的布局和触摸交互
- 保持功能完整性和操作便捷性
- 提供与桌面端一致的用户体验

## 二、设备与屏幕假设
- 目标设备：智能手机（iOS/Android）
- 屏幕宽度：375px ~ 414px（iPhone 6/7/8/SE到iPhone X/XS/11 Pro）
- 操作方式：单手触摸操作
- 网络环境：移动网络（3G/4G/5G）和Wi-Fi

## 三、整体信息架构
```
移动端北单筛选器（/m/beidan-filter）
├── 筛选条件页面（默认视图）
│   ├── 头部：标题 + 实时匹配数量
│   ├── 三维筛选区域（垂直堆叠）
│   │   ├── 实力等级差 ΔP（可折叠）
│   │   ├── 赢盘等级差 ΔWP（可折叠）
│   │   ├── 一赔稳定性 P-Tier（可折叠）
│   │   └── 其它条件（联赛、日期等）
│   ├── 策略选择区域
│   │   ├── 策略下拉选择器
│   │   └── 当前策略详情
│   └── 底部操作栏
│       ├── 重置筛选
│       ├── 应用筛选
│       └── 更多操作（导出、规则说明等）
├── 筛选结果页面
│   ├── 筛选结果卡片列表（上拉加载）
│   ├── 统计信息折叠面板
│   └── 比赛分析弹窗（全屏/底部抽屉）
└── 策略管理页面
    ├── 策略列表（简化视图）
    ├── 策略创建/编辑表单
    └── 策略操作（删除、应用等）
```

## 四、页面详细设计

### 4.1 筛选条件页面（主页面）
**布局结构：**
```
┌─────────────────────────┐
│ 三维精算筛选器          │ ← 固定顶部标题栏
│ 基于ΔP/ΔWP/P-Tier联动筛选│
├─────────────────────────┤
│ 实时匹配 125 场         │ ← 实时统计区域
├─────────────────────────┤
│ ▼ 实力等级差 ΔP         │ ← 可折叠筛选组
│   主客实力差分层        │
│   [ -3 ][ -2 ][ -1 ]    │
│   [  0 ][ +1 ][ +2 ]    │
│   [ +3 ]                │
├─────────────────────────┤
│ ▼ 赢盘等级差 ΔWP        │
│   盘路兑现力对撞        │
│   [ +4 ][ +3 ][ +2 ]    │
│   [ +1 ][  0 ][ -1 ]    │
│   [ -2 ][ -3 ][ -4 ]    │
├─────────────────────────┤
│ ▼ 一赔稳定性 P-Tier     │
│   正路可信度等级        │
│   [ S ][ A ][ B ]       │
│   [ B-][ C ][ D ]       │
│   [ E ]                 │
├─────────────────────────┤
│ 其它条件               │
│ ▢ 联赛筛选：[英超][西甲]│
│ ▢ date_time：[2024-01] │
│ ▢ 日期范围：01/01-01/31│
├─────────────────────────┤
│ 策略筛选               │
│ [选择策略] ▼           │ ← 下拉选择器
│ 当前策略：强势正路      │
│ ΔP: +2,+3 ΔWP: +3,+4   │
│ P-Tier: S,A            │
├─────────────────────────┤
│ [重置] [应用筛选] [···]│ ← 底部固定操作栏
└─────────────────────────┘
```

**交互细节：**
1. **顶部标题栏**：固定位置，显示主标题和副标题
2. **实时统计区域**：显示当前筛选条件下的匹配场次数量
3. **折叠筛选组**：默认展开第一个筛选组（ΔP），其他可折叠以节省空间
4. **选项按钮**：每个选项使用`el-checkbox-button`，最小尺寸44×44px
5. **策略选择**：使用Element Plus的`el-select`组件，支持搜索过滤
6. **底部操作栏**：固定位置，包含主要操作按钮


**交互细节：**
1. **返回按钮**：左上角返回筛选条件页面
2. **统计面板**：可折叠显示关键统计数据
3. **比赛卡片**：每场比赛显示为独立卡片，包含核心信息
4. **分析按钮**：点击进入比赛分析页面
5. **上拉加载**：替代桌面端的分页组件，支持无限滚动

## 五、交互流程设计

### 5.1 核心筛选流程
```
用户进入页面 → 查看默认筛选条件 → 调整三维筛选选项 → 
选择策略 → 点击"应用筛选" → 跳转至结果页面 → 
浏览比赛卡片 → 点击"分析"查看详情 → 返回结果页面
```

### 5.2 策略管理流程
```
用户点击"策略管理" → 进入策略列表页面 → 
点击"创建新策略" → 填写策略表单 → 保存策略 → 
返回策略列表 → 可选择应用或删除策略
```

### 5.3 数据导出流程
```
用户在结果页面 → 点击底部操作栏"···" → 
弹出操作菜单 → 选择"导出数据" → 选择格式（CSV/JSON/Excel）→ 
系统生成文件并触发下载 → 移动端下载管理器处理
```

## 六、触摸交互优化

### 6.1 触摸目标尺寸
- 所有可点击元素最小尺寸：44×44px
- 按钮内边距：12px~16px
- 选项间距：8px~12px

### 6.2 手势支持
1. **上拉加载**：在结果页面底部上拉触发加载更多
2. **滑动删除**：在策略列表左滑删除策略
3. **长按编辑**：长按策略卡片进入编辑模式
4. **下拉刷新**：在筛选条件页面下拉刷新实时数据

### 6.3 反馈机制
1. **点击反馈**：使用Element Plus的涟漪效果
2. **加载状态**：显示骨架屏或加载动画
3. **操作确认**：重要操作（删除）需要二次确认
4. **成功提示**：操作成功显示Toast提示

## 七、响应式断点设计
- `≥768px`：桌面端布局（保持现有）
- `375px~768px`：移动端布局（垂直堆叠）
- `≤375px`：紧凑布局（进一步优化间距）
