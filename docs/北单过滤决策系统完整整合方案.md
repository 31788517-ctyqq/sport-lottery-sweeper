# 北单过滤决策系统完整整合方案

## 1. 项目现状分析

### 1.1 现有北单过滤系统概述
当前项目已实现基于三维精算筛选器的北单过滤系统，核心功能包括：

**1.1.1 三维筛选维度**
- **实力等级差 (ΔP)**: [-3, -2, -1, 0, +1, +2, +3]，反映主客队基础实力对比
- **赢盘等级差 (ΔWP)**: [-4, -3, -2, -1, 0, +1, +2, +3, +4]，反映盘路表现和"性格"特征
- **一赔稳定性 (P-Tier)**: [S(P1), A(P2), B(P3), B-(P4), C(P5), D(P6), E(P7)]，反映正路可信度

**1.1.2 核心业务规则**
- **降级规则**: 方向背离时P级强制下移一级
- **排序规则**: P3场次必须排在P4之前，同P级按ΔWP降序，P7按Ssum升序

### 1.2 技术架构现状
#### 后端架构
- **API层**: FastAPI (`/api/v1/beidan-filter/*`)，包含实时计数、高级筛选、策略管理、数据导出等接口
- **服务层**: `BeidanDataService`（数据获取与转换）、`BeidanFilterEngine`（筛选逻辑）
- **数据层**: PostgreSQL数据库，`football_matches`表存储原始比赛数据
- **模型层**: `MatchModel`、`BeidanStrategy`（策略管理）

#### 前端架构
- **主组件**: `BeidanFilterPanel.vue`（三维筛选面板）
- **状态管理**: Vue 3 Composition API
- **UI框架**: Element Plus
- **功能模块**: 实时统计、策略保存/加载、结果展示、分析弹窗

### 1.3 已实现功能
1. 三维多选筛选（实力等级差、赢盘等级差、一赔稳定性）
2. 实时匹配场次计数

### 2.3 统计学过滤模块设计
#### 2.3.1 过滤条件定义
将用户提供的统计学条件映射到北单场景：

| 原条件 | 北单映射 | 说明 |
|--------|----------|------|
| 红球/蓝球个数 | 胜/负场次限制 | 限制投注方案中"胜"（主胜）或"负"（客胜）的场次数 |
| 和值范围 | 赔率和值范围 | 所有投注场次赔率（如主胜赔）的总和范围 |
| 奇偶比 | 赛果代码奇偶比 | 限制投注方案中赛果代码为奇数或偶数的比例 |

#### 2.3.2 数据模型扩展

#### 2.3.3 过滤算法
```python
class StatisticalFilterEngine:
    def apply_statistical_filters(self, matches: List[MatchModel], 
                                  filter_config: StatisticalFilter) -> List[MatchModel]:
        """
        应用统计学过滤
        """
        # 1. 胜/负场次限制（基于预测结果）
        filtered = self._filter_by_win_loss_count(matches, filter_config)
        
        # 2. 赔率和值范围
        filtered = self._filter_by_odds_sum(filtered, filter_config)
        
        # 3. 奇偶比过滤
        filtered = self._filter_by_odd_even_ratio(filtered, filter_config)
        
        return filtered
```

#### 2.3.4 实现细节
- **胜/负判断**: 基于三维筛选的`recommendation`字段（"重点关注"倾向主胜，"观望"倾向平局，"客队占优"倾向客胜）
- **赔率和值**: 使用`odds.homeWin`计算赔率和
- **奇偶判断**: 基于`lineId`末尾数字或`match_id`取模

### 2.5 自定义条件模块设计
#### 2.5.1 规则引擎架构
```
自定义条件引擎
    ├── 条件解析器（支持自然语言/公式）
    ├── 条件编译器（转为可执行代码）
    ├── 条件执行器（应用过滤）
    └── 条件验证器（语法/逻辑检查）
```

#### 2.5.2 支持的条件类型
1. **简单逻辑条件**
   - `主场球队包含"曼联" AND 实力等级>=+2`
   - `赔率差值 > 0.5 OR 稳定性等级为S`
   
2. **统计条件**
   - `最近5场胜率 > 60%`
   - `主队进球数平均值 >= 1.5`
   
3. **组合条件**
   - `(实力等级=+3 AND 赢盘等级>=+2) OR (稳定性=S AND 赔率<2.0)`

```python
# backend/app/services/enhanced_filter_engine.py
from typing import List, Dict, Any
from .filter_engine import BeidanFilterEngine, MatchModel

class EnhancedBeidanFilterEngine(BeidanFilterEngine):
    """增强版北单过滤引擎（集成智能过滤）"""
    
    def __init__(self):
        super().__init__()
        self.dan_tuo_filter = DanTuoFilter()
        self.statistical_filter = StatisticalFilterEngine()
        self.pattern_filter = PatternFilterEngine()
        self.custom_filter = CustomConditionEngine()
    
    def apply_enhanced_filters(self, matches: List[MatchModel], 
                               filter_config: Dict[str, Any]) -> List[MatchModel]:
        """
        应用增强过滤（包括智能过滤引擎）
        过滤顺序：三维筛选 → 胆拖过滤 → 统计学过滤 → 连号/重号过滤 → 自定义条件
        """
        # 1. 应用基础三维筛选
        filtered = super().apply_filters(matches, filter_config)
        
        # 2. 应用胆拖过滤
        if filter_config.get('dan_tuo_filter'):
            filtered = self.dan_tuo_filter.apply(filtered, filter_config['dan_tuo_filter'])
        
        # 3. 应用统计学过滤
        if filter_config.get('statistical_filter'):
            filtered = self.statistical_filter.apply(filtered, filter_config['statistical_filter'])
        
        # 4. 应用连号/重号过滤
        if filter_config.get('pattern_filter'):
            filtered = self.pattern_filter.apply(filtered, filter_config['pattern_filter'])
        
        # 5. 应用自定义条件
        if filter_config.get('custom_conditions'):
            filtered = self.custom_filter.apply(filtered, filter_config['custom_conditions'])
        
        return filtered
```

## 5. 兼容性说明

2. **数据兼容性**: 现有用户策略和数据不受影响
3. **界面兼容性**: 现有三维筛选面板保持原样，新增智能过滤为可选功能

### 5.2 配置迁移
- 新增智能过滤配置可选，默认关闭
- 用户可逐步迁移到新系统
- 提供配置导入/导出工具

### 5.3 性能影响
- 智能过滤作为可选模块，不使用时无性能影响
- 复杂过滤启用缓存机制
- 支持异步处理长时间运行的过滤任务
