# 日志管理模块测试用例总结

## 概述
本文档总结了为体育彩票扫盘系统中的日志管理模块编写的测试用例。日志管理模块包含5个主要子页面，提供了对系统各类日志的集中管理与分析功能。

## 日志管理模块结构
日志管理模块包含以下5个子页面：
1. **日志管理主页面** - 提供日志统计和导航入口
2. **系统日志页面** - 显示系统运行日志
3. **用户日志页面** - 显示用户活动日志
4. **安全日志页面** - 显示安全相关事件
5. **API日志页面** - 显示API访问日志

## 测试场景

### 真实用户操作场景
1. **日常监控场景**：管理员登录系统，查看系统日志，筛选错误级别的日志
2. **安全审查场景**：安全人员查看安全日志，查找可疑IP活动
3. **性能分析场景**：运维人员查看API日志，分析响应时间和状态码
4. **用户行为分析场景**：管理员查看用户日志，分析用户活动模式
5. **故障排查场景**：技术支持人员结合各种日志定位问题

### 测试数据结构
- **系统日志**：时间戳、级别、模块、消息、附加数据
- **用户日志**：时间戳、用户ID、模块、消息、IP地址、会话ID
- **安全日志**：时间戳、级别、模块、消息、IP地址、用户ID、附加数据
- **API日志**：时间戳、级别、请求路径、响应状态、耗时、IP地址、用户代理、会话ID、消息、附加数据

## API端点测试
- `GET /api/v1/admin/system/logs/db/system` - 获取系统日志
- `GET /api/v1/admin/system/logs/db/user` - 获取用户日志
- `GET /api/v1/admin/system/logs/db/security` - 获取安全日志
- `GET /api/v1/admin/system/logs/db/api` - 获取API日志
- `GET /api/v1/admin/system/logs/db/statistics` - 获取日志统计

## 过滤和分页参数
- `skip`: 跳过的记录数
- `limit`: 每页记录数
- `level`: 日志级别筛选
- `start_date`/`end_date`: 时间范围筛选
- `search`: 搜索关键词
- `user_id`: 用户ID筛选（仅用户日志）

## 运行测试
要运行所有测试，请执行以下命令：

```bash
node run_log_tests.js
```

或者分别运行不同类型测试：

```bash

# 集成测试
npm test -- tests/integration/log_api.test.js

# E2E测试
npm test -- tests/e2e/log_management.test.js
```
