# 北单三维筛选器后端接口路由草案

## 概述
本文档基于前端页面 `http://localhost:3000/admin/beidan-filter` 的实际功能，定义后端需要提供的数据接口路由规范。

## 基础信息
- **API 版本**: `/api/v1`
- **数据格式**: JSON
- **认证方式**: JWT Token (Bearer)
- **响应格式**: `{ "code": 200, "message": "success", "data": {...} }`

---

## 1. 实时数据接口

### 1.1 获取实时匹配场次数
**接口路径**: `GET /api/v1/beidan-filter/real-time-count`

**请求参数**: 无

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "matchCount": 156,
    "timestamp": "2025-01-15 14:30:25"
  }
}
```

**业务逻辑**:
- 返回当前系统中所有未开始的北单比赛场次数
- 数据实时更新，无需缓存

## 6. 高级筛选接口

### 6.1 执行高级筛选
**接口路径**: `POST /api/v1/beidan-filter/advanced-filter`

**请求参数**:
```json
{
  "threeDimensional": {
    "powerDifference": {
      "homeWeak": true,
      "homeBalanced": true,
      "homeStrong": true,
      "guestWeak": true,
      "guestBalanced": true,
      "guestStrong": true
    },
    "winPanDifference": 0,
    "sizeBallDifference": 0
  },
  "otherConditions": {
    "leagues": ["premier_league", "la_liga"],
    "dateTime": "2025-01-15 20:00",
    "dateRange": {
      "startDate": "2025-01-15",
      "endDate": "2025-01-20"
    },
    "strength": "balanced"
  },
  "sort": {
    "field": "match_time",
    "order": "asc"
  },
  "page": 1,
  "pageSize": 20
}
```

**响应示例**:

{

  "data": {
    "matches": [

        "id": 1001,
        "matchTime": "2025-01-15 20:00",

          "homeWin": 1.85,
          "draw": 3.40,
          "guestWin": 4.20
        },
        "strengthAnalysis": {
          "homeStrength": "偏强",
          "guestStrength": "均衡",
          "powerDifference": "主队占优"
        },
        "predictScore": "2:1",
        "recommendation": "重点关注"
      }
    ],
    "statistics": {
      "totalMatches": 15,

**业务逻辑**:
- 根据三维条件和其他条件进行复合筛选

## 9. 错误处理规范

### 错误响应格式
```json
{
  "code": 400,
  "message": "错误描述信息",
  "data": null
}
```

### 常见错误码
- `200`: 成功
- `400`: 请求参数错误
- `401`: 未授权访问
- `403`: 权限不足
- `404`: 资源不存在
- `500`: 服务器内部错误

---

## 10. 接口依赖关系

### 前端调用顺序
1. **页面加载时**:
   - `GET /api/v1/beidan-filter/date-time-options`
   - `GET /api/v1/beidan-filter/league-options`
   - `GET /api/v1/beidan-filter/strength-options`
   - `GET /api/v1/beidan-filter/win-pan-diff-options`
   - `GET /api/v1/beidan-filter/strategies`
   - `GET /api/v1/beidan-filter/example-strategies`

2. **用户操作时**:
   - 点击"获取实时数据": `GET /api/v1/beidan-filter/real-time-count`
   - 选择策略: `POST /api/v1/beidan-filter/advanced-filter`
   - 应用筛选: `POST /api/v1/beidan-filter/advanced-filter`
   - 导出数据: `POST /api/v1/beidan-filter/export/{format}`

3. **策略管理**:
   - 增删改查策略: 对应的策略管理接口

## 11. 性能要求

### 响应时间要求
- 选项类接口 (< 100ms): 日期时间、联赛、强度等级等
- 筛选接口 (< 500ms): 高级筛选，数据量 < 1000条
- 导出接口 (< 2s): 数据量 < 5000条

### 数据缓存策略
- 选项类接口: 可缓存5分钟
- 实时数据接口: 不缓存，实时查询
- 筛选接口: 根据数据更新频率设置缓存

---

## 12. 安全考虑

### 权限控制
- 所有接口需要JWT认证
- 策略管理接口需要管理员权限
- 导出接口限制单次最大数据量

### 数据验证
- 所有输入参数进行类型和范围验证
- SQL注入防护
- XSS攻击防护
