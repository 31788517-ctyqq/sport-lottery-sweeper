# 北单胜负平投注模拟器 — 详细开发计划

> 范围：后台“精算工具中心”新增子菜单“北单投注模拟”，实现投注方案管理、开奖匹配、串关模拟、盈亏计算与历史回看。
> 确认项：
> - 菜单为“精算工具中心”
> - 方案写入后端数据库，并与管理员账号关联
> - 支持串关（过关）
> - 要求“多串 N”或部分命中返奖规则
> - 赔率来源为数据库/接口
> - 结果匹配按场次序号即可
> - 期号下拉仅最近 5 期，不支持自定义期号
> - 方案命名无需用户输入（系统自动命名）
> - 允许删除方案
> - 不需要导出功能

## 1. 目标与交付物

### 1.1 目标
- 提供“北单投注模拟”页面：选择期号、查看比赛、选择赛果并生成投注方案。
- 从后端读取赔率与比赛信息，支持串关投注。
- 获取开奖数据并按“场次序号”匹配，计算每个方案的奖金与盈亏。
- 投注方案持久化到后端（与管理员账号关联），可查看历史方案与结果。

### 1.2 交付物
- 前端页面：`北单投注模拟`（含移动端适配）
- 菜单与路由：精算工具中心子菜单
- 后端 API：比赛/赔率、开奖结果、投注方案 CRUD、模拟计算
- 数据库表：投注方案与方案明细
- 端到端基本流程可用（创建方案 → 拉取开奖 → 计算盈亏 → 历史回看）

## 2. 需求分解

### 2.1 页面功能
- 期号选择（最近 5 期，默认最新期）
- 比赛列表：场次序号、主客队、开赛时间、赔率（胜/平/负）
- 赛果选择与投注金额输入
- 支持选择多场进行串关（过关）
- 方案保存（系统自动命名，默认时间戳/期号组合）
- 结果模拟：
  - 拉取开奖数据
  - 按场次序号匹配
  - 计算命中场次、中奖金额、盈亏
- 历史方案查看：筛选期号、查看方案明细与计算结果

### 2.2 计算规则（串关与多串 N）
- 过关基础：方案包含多场，若选择“全串关”，需全中才中奖。
- 多串 N：允许选择串关类型（如 2 串 1、3 串 1、4 串 1 等），将所选场次组合成若干子注。
- 部分命中返奖：当未全中但命中某些子注时，仍按子注返奖。
- 子注赔率 = 该子注所含场次赔率乘积
- 子注中奖金额 = 子注投注金额 × 子注赔率
- 总中奖金额 = 所有中奖子注金额之和
- 盈亏 = 总中奖金额 − 方案投注总金额
- 投注金额分配：默认均分到所有子注（后续可扩展为自定义分配）

## 3. 后端设计

### 3.1 新增数据表（建议）

1) `beidan_betting_schemes`
- `id` (PK)
- `admin_user_id` (FK)
- `expect` (期号)
- `name` (方案名称，系统自动生成)
- `stake` (投注金额)
- `pass_type` (串关类型，如 2x1/3x1/4x1/全串关)
- `split_mode` (金额分配方式，默认均分)
- `total_odds` (全串关总赔率，便于展示)
- `status` (pending/won/lost)
- `win_amount` (中奖金额)
- `profit` (盈亏)
- `created_at` / `updated_at`

2) `beidan_betting_scheme_items`
- `id` (PK)
- `scheme_id` (FK)
- `match_seq` (场次序号)
- `home_team` / `away_team`
- `selected_result` (win/draw/lose)
- `odds` (所选赔率)
- `match_time`
- `result` (开奖结果：win/draw/lose)

### 3.2 API 设计（建议）

#### 3.2.1 比赛与赔率
- `GET /api/v1/beidan/matches?expect=26025`
  - 返回期号比赛列表（含场次序号、队伍、时间、胜平负赔率）

#### 3.2.2 开奖结果
- `GET /api/v1/beidan/results?expect=26025`
  - 后端代理抓取 `https://zx.500.com/zqdc/kaijiang.php?playid=1&expect=26025`
  - 解析并返回标准化结构（match_seq -> result）

#### 3.2.3 投注方案 CRUD
- `POST /api/v1/beidan/betting-schemes`（包含 `pass_type`、`stake`、`split_mode`、所选场次与赛果）
- `GET /api/v1/beidan/betting-schemes?expect=26025&page=1&page_size=20`
- `GET /api/v1/beidan/betting-schemes/{id}`
- `DELETE /api/v1/beidan/betting-schemes/{id}`

#### 3.2.4 方案计算
- `POST /api/v1/beidan/betting-schemes/{id}/simulate`
  - 调用开奖结果，写回 status / win_amount / profit

> 权限：管理员 JWT 鉴权，关联当前用户。

## 6. 风险与注意事项
- 500.com 页面结构变动可能导致解析失败 → 后端需容错与日志记录。

- 大量方案历史查询需分页。
