- ✗ 无法删除用户

```
离职流程: 权限由合规负责人接管
系统角色: ✓ 系统内置（不可删除）
```

- 查看所有操作日志

- ✗ 无法修改任何数据


- ✗ 无法修改任何内容

## 🔧 常见操作指南

### 1️⃣ 创建新角色

**步骤**:
1. 进入 `/admin/users/roles`
2. 点击左上角"新建角色"按钮
3. 填写表单信息：
   - **角色名称**: 自定义（必填）
   - **角色等级**: 选择1-5（必填）
   - **角色描述**: 说明该角色的用途（可选）
   - **系统内置**: 勾选则不可删除（可选）
4. 在权限树中勾选需要的权限
5. 点击"确定"保存

**示例**: 创建"内容审核员"
```
角色名称: 内容审核员
等级: L2 (运营员)
描述: 专门负责内容审核和质量控制
勾选权限: admin:user:view, task:view, task:execute, data:query, data:edit
```

### 2️⃣ 编辑角色权限

**步骤**:
1. 在右侧列表中选择要编辑的角色
2. 右侧"权限配置"区域会显示该角色的权限
3. 在权限树中 勾选/取消勾选 权限
4. 点击"保存权限"按钮
5. 确认保存（刷新后生效）

**提示**:
- 勾选父权限会自动选中子权限
- "重置"按钮可以撤销未保存的修改
- 修改系统角色的权限需要超级管理员权限

### 3️⃣ 分配用户到角色

在用户管理页面（`/admin/users/list`）:
1. 找到要修改的用户
2. 点击"编辑"按钮
3. 选择"角色选择"下拉框
4. 选择对应的角色（会自动获得该角色的所有权限）
5. 点击保存

### 4️⃣ 删除自定义角色

**步骤**:
1. 在角色列表中找到要删除的角色
2. 右下角点击"删除"按钮
3. 确认删除对话框

**注意**:
- ⚠️ 系统内置角色（L1-L5）无法删除
- ⚠️ 如果有用户分配了该角色，需要先重新分配用户

**检查非系统角色**:
- 只有"系统"标签没有勾选的角色才能删除
- 确认该角色未被任何用户使用

### 5️⃣ 重新初始化5层角色

如果系统角色被误删或损坏，可以重新初始化：


## ⚠️ 安全最佳实践

### 1. 超级管理员数量限制
```
推荐: 不超过2人
分别由: 
- 技术负责人
- 业务负责人
负责管理
```

### 2. 权限最小化原则
```
❌ 不要: 
- 项目初期就给运营员管理员权限
- 临时工给超管权限

✓ 应该: 
- 只分配必要的权限
- 定期审查和清理权限
- 使用临时权限有期限
```

### 3. 权限變更通知
```
当修改用户角色时:
1. 用户会在下次登录时获得新权限
2. 建议通知用户权限变更内容
3. 记录变更原因（在操作日志中）
```

### 4. 离职处理
```
员工离职时:
1. ❌ 不能直接删除用户（保留日志）
2. ✓ 应该禁用用户账户 (status = locked)
3. ✓ 清理临时权限和IP限制
4. ✓ 交接相关权限给替代人员
5. ✓ 生成离职清单报告
```

## 🆘 常见问题

### Q1: 新创建的角色，用户登录后没有权限生效？
**A**: 权限生效需要重新登录。用户必须新开浏览器标签页或完全退出重新登录。

### Q2: 我是管理员，为什么无法删除某个角色？
**A**: 系统内置角色（L1-L5）无法删除。查看角色列表，如果有"系统"标签说明无法删除。

### Q3: 如何临时给某个运营员提升为管理员权限？
**A**: 目前版本一个用户一个角色。建议创建"临时管理员"角色，使用完后再改回原角色。

### Q4: 为什么权限树显示空白？
**A**: 检查API是否返回权限数据：
```bash
curl http://localhost:8001/api/v1/permissions/tree
```

### Q5: 能否设置权限有效期？
**A**: 当前版本不支持。可以创建"临时"角色，在Google日历中设置提醒。

### Q6: 删除了自定义角色，现在有用户无法登录？
**A**: 需要管理员手动重新分配用户到其他角色。建议从系统备份或操作日志恢复。
