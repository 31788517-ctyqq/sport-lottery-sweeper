- 后端地址：`http://127.0.0.1:8000`
- 执行日期：____年__月__日
- 执行人：________
- 环境：`预发 / 生产镜像预演`

## 一、回归前准备

1. 启动后端并确认健康：
   `GET http://127.0.0.1:8000/health/live` 返回 `200`。
2. 使用管理员账号登录后台。
3. 打开浏览器开发者工具，清空 Console 与 Network。
4. 强制刷新页面（`Ctrl + F5`）。

## 二、数据层点检（接口）

1. 校验供应商数量接口：
   `GET /api/v1/llm-providers/count` 返回 `200` 且 `count > 0`。
2. 校验供应商列表接口：
   `GET /api/v1/llm-providers/?skip=0&limit=20` 返回 `200` 且数组非空。
3. 校验历史恢复 4 个供应商存在：
   必须能检索到以下名称：
   - `OpenAI Official`
   - `QWEN`
   - `MY_ZP_BIGMODLE`
   - `Test Frontend Provider`
4. 校验接口无异常码：
   页面加载和操作期间，Network 不应出现 `401/422/500`。

## 三、渲染层点检（页面）

1. 页面主卡片、筛选区、表格正常显示，无空白或错位。
2. 表格列展示正确：
   - 供应商名称
   - 类型
   - 启用状态
   - 健康状态
   - 优先级
   - 模型、请求数、成本等
3. 操作列按钮完整可见且不换行异常：
   - 测试
   - 启用/禁用
   - 编辑
   - 删除
4. 分页组件可见，页码切换后表格正常刷新。

## 四、逻辑层点检（功能）

1. 搜索：
   输入关键字 `OpenAI` 点击搜索，结果数量变化符合预期。
2. 重置：
   点击重置后回到全量列表。
3. 新增：
   新增一个临时供应商（建议命名 `UAT_REMOTE_时间戳`），保存成功。
4. 编辑：
   修改临时供应商描述并保存，刷新后值保持。
5. 启用/禁用：
   对临时供应商执行状态切换，状态标签正确变化。
6. 连通性测试：
   点击“测试”，接口返回成功或明确错误提示（不允许静默失败）。
7. 删除：
   删除临时供应商，刷新后不可见。

## 五、重点验收项（本次修复相关）

1. 页面请求路径应为 `/api/v1/llm-providers/*`。
2. 不应再出现“数量有值但列表为空”的情况。
3. 恢复的 4 个供应商应可在页面直接看到或通过搜索检索到。

## 六、回收与留痕

1. 删除所有本次回归创建的临时供应商（如 `UAT_REMOTE_*`）。
2. 导出并保存以下证据：
   - 页面截图（全量列表 + 搜索结果 + 新增编辑记录）
   - 关键接口返回截图
   - Console/Network 无严重错误截图
3. 回归结果结论（勾选）：
   - [ ] 通过（可发布）
   - [ ] 不通过（阻断发布）

## 七、失败处理建议

1. 若列表为空但 `count > 0`：优先检查请求是否命中 `/api/v1/llm-providers/`。
2. 若 401：检查本地登录态 token 是否过期。
3. 若 422/500：记录请求参数与响应体，提交后端排查。
