# 比赛数据管理菜单端到端测试用例

## 测试概述

本测试用例涵盖了比赛数据管理菜单的5个子菜单页面的完整端到端功能测试，包括所有操作按钮及功能。

## 测试环境准备

1. 确保后端服务已启动
2. 确保数据库已正确初始化
3. 确保拥有管理员账户
4. 确保前端应用已启动

## 测试子菜单页面

1. 联赛管理 (League Management)
2. 比赛管理 (Match Management)
3. 赔率管理 (Odds Management)
4. 竞彩赛程 (Lottery Schedule)
5. 北单赛程 (Beidan Schedule)

## 综合测试场景

### 场景 1: 完整比赛数据流程测试
**目标**: 验证从联赛创建到比赛、赔率、赛程管理的完整流程
**步骤**:
1. 在联赛管理页面创建一个新的联赛
2. 在比赛管理页面为该联赛添加比赛
3. 在赔率管理页面为比赛设置赔率
4. 在竞彩赛程页面查看赛程
5. 在北单赛程页面发布相关赛程
6. 验证所有数据正确关联

### 场景 2: 数据更新流程测试
**目标**: 验证数据更新在所有相关模块中的同步
**步骤**:
1. 修改联赛信息
2. 验证比赛管理页面中联赛信息同步更新
3. 验证赔率管理页面中联赛信息同步更新
4. 验证赛程管理页面中联赛信息同步更新

### 场景 3: 数据删除流程测试
**目标**: 验证数据删除对相关模块的影响
**步骤**:
1. 删除某个联赛
2. 验证该联赛下的比赛是否被正确处理
3. 验证相关赔率是否被正确清理
4. 验证相关赛程是否被正确处理

## 测试验收标准

- 所有测试用例通过率 >= 95%
- API响应时间 < 2秒
- 前端页面加载时间 < 3秒
- 统计数据准确性 100%
- 所有功能按钮响应正常
- 数据在各模块间正确同步

1. 使用Python requests库进行API调用测试
2. 使用Selenium进行前端UI自动化测试
3. 每个测试用例包含详细的日志输出
